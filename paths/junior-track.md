# Путь Junior System Analyst

Этот трек проведёт вас от базовых понятий до работы с требованиями и понимания REST API.
Рекомендованная последовательность:

1. **Клиент-серверная архитектура** — фундаментальное понятие
2. **Что такое протокол** — общее понимание
3. **Что такое требование** — основы работы аналитика
4. **Коммуникация со стейкхолдерами** — как собирать требования
5. **User Stories** — как формализовать требования
6. **BDD-сценарии** — формальные критерии приёмки
7. **BPMN** — моделирование процессов
8. **REST API** — практическая интеграция
9. **OpenAPI** — документирование API
