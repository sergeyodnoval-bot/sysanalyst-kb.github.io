# Confluence

Confluence — это вики-система для ведения проектной документации от Atlassian. Если Jira отвечает на вопрос «кто что делает», то Confluence отвечает на вопрос «как это работает и почему».

## Для чего используется

- Ведение документации проекта (SRS, architecture decision records)
- Хранение результатов интервью и аналитики
- База знаний для команды (onboarding, регламенты, инструкции)
- Совместное редактирование документов в реальном времени
- Интеграция с Jira — привязка задач к страницам документации

## Ключевые концепции

### Space (пространство)

Отдельный раздел для проекта или команды. Внутри space — иерархия страниц.

### Page (страница)

Базовая единица контента. Поддерживает текст, таблицы, диаграммы, вложения, макросы.

### Template (шаблон)

Готовая структура страницы для типовых документов: meeting notes, decision log, technical specification.

### Macro (макрос)

Динамический элемент на странице: диаграмма (Draw.io, Mermaid), Jira-фильтр, календарь, опрос.

### Whiteboard (доска)

Виртуальная доска для мозговых штурмов и диаграмм реального времени.

## Типовые страницы аналитика

### SRS (Software Requirements Specification)

Структура:
1. Цель и область применения
2. Общая архитектура
3. Функциональные требования (со ссылками на Jira-задачи)
4. Нефункциональные требования
5. Ограничения и допущения
6. Глоссарий

### Результаты интервью

1. Дата, участники, контекст
2. Ключевые проблемы (прямые цитаты)
3. Предложенные решения
4. Договорённости и action items

### Decision Log

1. Дата решения
2. Контекст — почему этот вопрос возник
3. Рассмотренные варианты
4. Выбранное решение и обоснование
5. Кто принимал решение

## Для кого этот инструмент

Confluence — общее пространство команды:

- **System Analyst** — пишет SRS, фиксирует интервью, описывает архитектуру
- **Developer** — читает требования, оставляет комментарии
- **Project Manager** — документирует планы и решения
- **Designer** — выкладывает прототипы и UI Kit
- **New team members** — изучают документацию для онбординга

## Преимущества

- Тесная интеграция с Jira — можно вставлять фильтры и ссылаться на задачи
- Совместное редактирование в реальном времени
- Шаблоны для типовых документов
- Поддержка Draw.io и Mermaid для диаграмм
- Версионирование страниц — всегда видно, кто и когда менял

## Недостатки

- Визуальный редактор — неудобно вставлять код и форматировать сложные таблицы
- Поиск — не самый быстрый и точный
- Платная лицензия (как и Jira)
- Склонность к «кладбищу страниц» — неактуальная документация копится годами

## Как начать

1. Создайте space для вашего проекта
2. Выберите шаблон для первой страницы
3. Напишите краткий обзор проекта
4. Создайте иерархию: документы, встречи, принятые решения
5. Пригласите команду и настройте права доступа
