# Figma

Figma — это инструмент для дизайна интерфейсов и прототипирования. Главное отличие от конкурентов — **работает в браузере** и не требует установки (хотя есть и десктоп-клиент). Всё хранится в облаке, командная работа — открыть ссылку и редактировать одновременно.

## Для чего используется

- **Дизайн экранов** — как будет выглядеть страница или приложение
- **Интерактивные прототипы** — можно кликать и переходить между экранами
- **Design Systems** — библиотеки компонентов (кнопки, поля ввода, иконки), переиспользуемые во всех макетах
- **Передача в разработку** — разработчик смотрит размеры, отступы, цвета, экспортирует ресурсы
- **Wireframes** — черновые схемы без дизайна, только структура

## Ключевые понятия

| Понятие | Описание |
|---------|----------|
| **Frame** | Экран или отдельное окно (аналог артборда) |
| **Layer** | Слой — любой элемент внутри Frame |
| **Component** | Переиспользуемый элемент (кнопка, карточка). Изменение компонента обновляет все его экземпляры |
| **Auto Layout** | Автоматическое выравнивание — резиновые блоки, подстраивающиеся под содержимое |
| **Prototype** | Режим связывания экранов переходами (по клику, по ховеру) |
| **Dev Mode** | Режим для разработчиков — размеры, отступы, CSS-свойства |

## Чем Figma лучше аналогов

- **Бесплатно для личного использования** — неограниченное количество файлов
- **Работает в любом браузере** — не нужен мощный ПК
- **Реальное время** — 10 человек могут править один файл, изменения видны мгновенно
- **Плагины** — от интеграции с Jira до генерации JSON из макетов
- **Dev Mode** — показывает CSS, SwiftUI, Compose-код напрямую

## Сравнение с альтернативами

| Критерий | Figma | Sketch | Adobe XD |
|----------|-------|--------|----------|
| Платформа | Браузер + десктоп | Только macOS | Windows + macOS |
| Совместная работа | Встроена (реальное время) | Через сторонние сервисы | Встроена (облако) |
| Цена | Бесплатно / $12 мес | $10/год (подписка) | Бесплатно / $10 мес |
| Плагины | Более 1000 | Много, но только macOS | Ограничено |
| Передача разработчику | Dev Mode (бесплатно) | Zeplin / Avocode | Встроено |

**Когда выбирать Figma:** вы работаете в браузере, нужна совместная работа, бюджет ограничен.

**Когда выбирать Sketch:** команда на macOS, нужна проверенная экосистема с многолетней историей.

**Когда выбирать Adobe XD:** команда уже использует Adobe-экосистему (Photoshop, Illustrator).

## Как аналитику работать с Figma

Аналитик обычно не рисует чистовой дизайн, но постоянно работает с макетами:

### 1. Посмотреть макет
- Откройте ссылку-приглашение (Figma отправляет `figma.com/file/...`)
- Включите **Dev Mode** (кнопка в правом верхнем углу) — увидите размеры, отступы, цвета
- Используйте **Inspector** (правая панель) чтобы посмотреть точные значения

### 2. Сделать простой wireframe
- Создайте Frame размером с экран (например, `1440×900` для десктопа)
- Накидайте прямоугольники — это будут блоки (шапка, меню, контент)
- Подпишите каждый слой (двойной клик по имени слоя)
- Используйте стрелки-комментарии (Shift+S) для указания связей

### 3. Оставить комментарий
- Выделите элемент и нажмите `C` — появится поле для комментария
- Можно упомянуть дизайнера через `@имя`
- К каждому комментарию можно прикрепить скриншот

### 4. Экспортировать для документации
- Кликните на нужный элемент
- В правой панели найдите **Export** (или `Ctrl+Shift+E`)
- Выберите PNG / SVG / PDF
- Используйте плагин **Figma to Google Slides** для презентаций

## Когда использовать, когда НЕТ

| Использовать | НЕ использовать |
|---|---|
| Просмотр макетов и прототипов | Вместо draw.io для BPMN/UML (нет нотаций) |
| Простые wireframes для обсуждения | Для Enterprise Architect-диаграмм |
| Совместная работа над визуальными спецификациями | Для текстовой документации (Confluence) |
| Прототипирование потока экранов | Для долгосрочного хранения (нет версионирования как в Git) |

## Как начать

1. Зарегистрируйтесь на [figma.com](https://www.figma.com/) (бесплатно)
2. Откройте **Community** — там тысячи готовых шаблонов (User Flow, Wireframe Kit)
3. Создайте Frame и попробуйте добавить прямоугольники и текст
4. Переключитесь в **Prototype** и свяжите два экрана
5. Примите приглашение от дизайнера в реальном проекте и включите Dev Mode
