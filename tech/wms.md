WMS (Warehouse Management System) — система управления складом: приёмка, размещение, сборка, отгрузка, инвентаризация.

**Ключевые функции WMS:**
- **Receiving** — приёмка товара (сканирование, QC)
- **Putaway** — размещение по ячейкам (по правилам: ABC, FEFO, FIFO)
- **Picking** — сборка заказов (поштучно, волнами, кластерами)
- **Packing** — упаковка, взвешивание, маркировка
- **Shipping** — сортировка, загрузка, передача TMS
- **Inventory** — инвентаризация (циклическая, полная)
- **Returns** — обработка возвратов

**Для аналитика:** WMS оперирует ячейками (bin locations), а не просто «складом». Заказ из OMS приходит в WMS как задание на сборку. WMS управляет маршрутом сборщика по складу. Интеграция — через MQ (Kafka/RabbitMQ) для асинхронной синхронизации статусов и остатков.

**Основные сущности WMS:**
- **Location** — ячейка хранения (ряд-стеллаж-полка)
- **SKU** — товарная позиция
- **Lot** — партия (срок годности, серийный номер)
- **Task** — задание сборщику
- **Wave** — волна (группа заказов для оптимизации маршрута)
