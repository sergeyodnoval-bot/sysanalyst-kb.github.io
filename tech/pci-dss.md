PCI DSS (Payment Card Industry Data Security Standard) — стандарт безопасности данных платёжных карт, обязательный для всех, кто обрабатывает, хранит или передаёт данные карт (PAN, CVV, срок действия).

**Ключевые требования (PCI DSS v4.0):**

| Требование | Пример |
|-----------|--------|
| 1. Защищённая сеть | Firewall, сегментация |
| 2. Безопасные конфигурации | Безопасные настройки ПО |
| 3. Защита данных | Шифрование PAN, не хранить CVV |
| 4. Шифрование передачи | TLS 1.2+ |
| 5. Защита от вредоносного ПО | Антивирус, EDR |
| 6. Безопасная разработка | SAST, DAST, code review |
| 7. Контроль доступа | RBAC |
| 8. Аутентификация | MFA |
| 9. Физическая безопасность | Контроль доступа к серверам |
| 10. Мониторинг | SIEM, алерты |
| 11. Тестирование безопасности | Pen-test, ASV-сканирование каждые 90 дней |
| 12. Политики безопасности | Документация, обучение |

**Для аналитика:** важно понимать scope (какие системы входят в PCI DSS) и SAQ (Self-Assessment Questionnaire). Если система принимает/обрабатывает карты — применяется PCI DSS, что влияет на архитектуру (шифрование, логирование без PAN, сегрегация CDE).

**SAQ уровни:** SAQ A (e-commerce без хранения), SAQ D (самый полный — мерчанты, шлюзы).

**SAQ** — упрощённое анкетирование для малого бизнеса, **ROC** (Report on Compliance) — полноценный аудит для крупных (Level 1 — > 6 млн транзакций/год).
