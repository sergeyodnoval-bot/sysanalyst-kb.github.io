:::info TL;DR
ISO 20022 — международный стандарт для электронного обмена данными между финансовыми институтами. Заменяет старые форматы (MT — в SWIFT, UFE/STM — в национальных системах). Это XML-based стандарт с расширяемой схемой, поддерживающий rich data (структурированные адреса, реквизиты, назначение платежа).
:::

## Для чего используется

- Межбанковские переводы (SCT, SCT Inst)
- Платёжные поручения клиентов
- Выписки по счетам
- Уведомления об операциях
- Кредитные и дебетовые ноты
- Отчёты по транзакциям

## Ключевые концепции

- **Business model** — описание бизнес-процессов (UML-диаграммы)
- **Logical message** — XML/Schema-определение сообщения, независимое от транспорта
- **Physical representation** — сериализация: XML (ISO 20022 XML) или ASN.1 (для SWIFT)

**Популярные сообщения ISO 20022 для аналитика:**

| Сообщение | Назначение |
|-----------|-----------|
| `pacs.008` | SEPA Credit Transfer — перевод средств |
| `pacs.009` | Financial Institution Transfer — межбанковский перевод |
| `pacs.002` | Status — статус обработки |
| `camt.053` | Bank-to-Customer Statement — выписка |
| `camt.054` | Credit/Debit Notification — уведомление об операции |
| `pain.001` | Customer Credit Transfer — платёж от клиента |
| `pain.002` | Status — статус платежа |

## Когда использовать

- Интеграция с европейскими банками (SEPA)
- Миграция с SWIFT MT на MX
- Новые платёжные системы (FedWire, TARGET2, EBA Clearing)
- B2B-платежи с rich data

## Когда НЕ использовать

- Карточные транзакции (POS/ATM) — используйте ISO 8583
- Устаревшие банки без поддержки ISO 20022
- Сверхвысокочастотные торговые операции (latency sensitive)

## Альтернативы

| Стандарт | Описание |
|----------|----------|
| **ISO 8583** | Карточные транзакции, бинарный формат |
| **SWIFT MT** | Legacy межбанковский формат |
| **EDI** | Корпоративный электронный документооборот |
| **JSON API (Berlin Group)** | REST-альтернатива для Open Banking |

## Как начать

1. Изучить каталог сообщений на iso20022.org
2. Определить нужные message types (pacs, camt, pain)
3. Скачать XSD-схемы для нужной версии
4. Настроить XML-валидацию в проекте
5. Пройти certification (SWIFT/SEPA)
