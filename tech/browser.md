# Веб-браузер

Браузер — это **клиент** в клиент-серверной архитектуре: программа, которая отправляет HTTP-запросы, получает ответы и отображает веб-страницы.

Для системного аналитика браузер — не просто средство просмотра сайтов, а **основной инструмент отладки и исследования** веб-систем. Инструменты разработчика (DevTools) позволяют видеть, какие запросы отправляет страница, какие данные приходят с сервера, как работает API.

## Основные браузеры — сравнение

| Браузер | Движок | Плюсы | Минусы |
|---------|--------|-------|--------|
| **Chrome** | Blink (V8) | Лучшие DevTools, огромное расширение, рынок > 65% | Жрёт память, слежка от Google |
| **Firefox** | Gecko (SpiderMonkey) | Лучшая приватность, открытый код, хорошие DevTools | Меньше расширений, меньше рынка |
| **Safari** | WebKit (JavaScriptCore) | Энергоэффективность на Mac, встроен в iOS | DevTools слабее, только Apple |
| **Edge** | Blink (V8) | Встроен в Windows, те же DevTools что Chrome | Меньше расширений |

Для аналитика **Chrome или Edge** — оптимальный выбор: самые мощные DevTools и наибольшая совместимость.

## Инструменты разработчика (DevTools)

DevTools открываются по `F12` или `Ctrl+Shift+I` в любом браузере. Вот что реально нужно аналитику:

### Network (Сеть)

Самая важная вкладка для аналитика. Показывает **все HTTP-запросы**, которые делает страница.

**Как использовать:**
1. Откройте DevTools → **Network**
2. Найдите интересующий запрос (по URL, методу, статусу)
3. Смотрите:
   - **Headers** — что отправлено (метод, заголовки, cookies)
   - **Payload / Request** — тело запроса (JSON, Form Data)
   - **Response** — что вернул сервер (JSON, HTML, ошибка)
   - **Timing** — сколько времени занял запрос
4. Фильтруйте по типу: `XHR` / `Fetch` (только API-запросы), `Doc` (страницы), `Img` (картинки)

**Пример:** Вы тестируете форму регистрации. Откройте Network, заполните форму и нажмите «Отправить». Вы увидите POST-запрос с телом формы и ответ сервера с результатом. Это позволяет проверить, какие данные реально уходят на сервер.

### Console (Консоль)

Показывает:
- Ошибки JavaScript (например, «Failed to fetch»)
- Предупреждения (deprecated API, проблемы безопасности)
- Результаты `console.log()` — если разработчик оставил отладку
- Можно выполнить произвольный JavaScript (например, `fetch('/api/users').then(r => r.json())`)

### Elements (Элементы)

HTML-структура страницы. Полезно, когда нужно:
- Понять, как устроена страница (какие блоки, иерархия)
- Скопировать CSS-свойства (размеры, цвета, отступы) для документации
- Временно изменить текст на странице (для макета документации)

### Application (Хранилище)

Показывает:
- **Local Storage / Session Storage** — что хранит сайт на вашем компьютере
- **Cookies** — какие куки установлены, их значения и срок жизни
- **IndexedDB** — клиентская база данных

Полезно для анализа: как сайт запоминает пользователя, какие данные хранит на клиенте.

## Как аналитик использует браузер

### 1. Исследование существующей системы
- Откройте DevTools → Network → перезагрузите страницу
- Смотрите все запросы: какие API вызываются, какие данные передаются
- Сохраните HAR-файл (Export HAR) — полный лог всех запросов для документации

### 2. Проверка требований
- В спецификации написано «при ошибке валидации показывать сообщение» — откройте DevTools, отправьте невалидные данные, смотрите Response в Network
- Проверьте, что API возвращает правильный HTTP-статус (400 vs 500)

### 3. Отладка интеграций
- Проверьте CORS-ошибки (консоль покажет красную ошибку)
- Проверьте заголовки авторизации в запросах
- Посмотрите, какие cookies отправляются с запросом

### 4. Составление документации
- Используйте **Network** для сбора реальных примеров запросов/ответов
- Используйте **Elements** для скриншотов элементов интерфейса
- Экспортируйте HAR для технического аудита

## Когда какой браузер выбирать

| Сценарий | Браузер |
|----------|---------|
| Повседневная работа | Chrome / Edge |
| Проверка под iOS | Safari |
| Тестирование приватности | Firefox |
| Отладка API | Chrome / Edge (лучшие Network-инструменты) |
| Нужны расширения для аналитика | Chrome (Postman Interceptor, JSON Viewer, Window Resizer) |

## Как браузер исполняет код — кратко

1. Парсит HTML → строит DOM-дерево
2. Загружает CSS → строит CSSOM
3. Объединяет в Render Tree
4. Выполняет JavaScript (движок V8 в Chrome, SpiderMonkey в Firefox)
5. Отрисовывает страницу (GPU-ускорение)

Понимание этого цикла помогает аналитику формулировать нефункциональные требования: «страница должна загружаться не более 2 секунд» — это ограничение и на сервер, и на браузерный рендеринг.
