# Event Storming

Event Storming — это воркшоп-техника для моделирования бизнес-процессов через события. Придумал Alberto Brandolini. Это не диаграмма, а формат встречи, где вся команда наклеивает стикеры на огромную доску.

## Идея

Вместо того чтобы рисовать архитектуру «сверху вниз», Event Storming начинается с событий — того, что уже произошло в бизнесе. Команда постепенно добавляет команды, агрегаты, роли и ограничения.

## Стадии Event Storming

### 1. Chaos (хаос)

Участники наклеивают оранжевые стикеры — **события** (Event). Каждое событие — факт в прошлом: `ЗаказСоздан`, `ТоварОтгружен`, `СчётВыставлен`.

Правила: глагол в прошедшем времени, один стикер — одно событие, не обсуждаем правильность.

### 2. Timeline

События располагаются на временной шкале слева направо. Убираются дубли, добавляются пропущенные события.

### 3. Pain points

Красные стикеры — «узкие места», проблемы, вопросы. Команда визуализирует, где процесс болит.

### 4. Commands (команды)

Голубые стикеры — **команды** (Command), которые вызывают события. Кто-то делает `СоздатьЗаказ` → возникает `ЗаказСоздан`.

### 5. Aggregates (агрегаты)

Жёлтые стикеры — **агрегаты** (сущности, над которыми выполняются команды). Группируют команды и события в бизнес-объекты.

### 6. Bounded Contexts

Большие рамки — **bounded context** (границы модели из DDD). `Заказы`, `Платежи`, `Доставка` — каждый контекст имеет свой язык и модель.

### 7. Policies (политики)

Лиловые стикеры — автоматические реакции. `Когда ЗаказОплачен → запустить Доставку`.

## Цвета стикеров

| Цвет | Элемент | Пример |
|------|---------|--------|
| Оранжевый | Event | `ЗаказСоздан` |
| Голубой | Command | `СоздатьЗаказ` |
| Жёлтый | Aggregate | `Заказ` |
| Зелёный | Actor/User | `Покупатель` |
| Лиловый | Policy | `Когда ЗаказОплачен → Отгрузить` |
| Красный | Pain point | `Долгая верификация` |

## Роль аналитика

Аналитик обычно фасилитирует Event Storming:

- Готовит доску (Miro / физическая)
- Приглашает экспертов (PO, разработчики, бизнес)
- Объясняет правила
- Следит за таймингом
- Фотографирует результат

После воркшопа аналитик:
- Оцифровывает доску
- Определяет bounded context для микросервисов
- Описывает события в деталях
- Использует результат для ADR
