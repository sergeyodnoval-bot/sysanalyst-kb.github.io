# Cost-Benefit Analysis (CBA)

Cost-Benefit Analysis — метод сравнения затрат на реализацию решения с выгодами от его внедрения. Позволяет ответить на вопрос: **стоит ли это делать?**

## Зачем аналитику CBA

- Обосновать выбор одного решения из нескольких альтернатив
- Показать заказчику ценность (ROI) предлагаемого функционала
- Ответить на вопрос «зачем нам это нужно?» на языке бизнеса
- Приоритизировать инициативы в условиях ограниченного бюджета

## Структура CBA

### 1. Costs (Затраты)

Прямые:
- Разработка (человеко-часы × ставка)
- Инфраструктура (серверы, лицензии, облако)
- Внедрение (миграция, обучение, поддержка)
- Интеграция (доработка смежных систем)

Косвенные:
- Простой системы во время внедрения
- Снижение продуктивности команды на период адаптации
- Риск неудачи (вероятность × стоимость)

### 2. Benefits (Выгоды)

Количественные (измеримые):
- Рост выручки (новые клиенты, увеличение conversion rate)
- Снижение затрат (автоматизация ручного труда)
- Экономия времени (операция сократилась с 5 минут до 30 секунд)
- Снижение ошибок (меньше переделок, возвратов)

Качественные (субъективные):
- Удовлетворённость пользователей
- Соответствие регуляторным требованиям
- Конкурентное преимущество
- Повышение предсказуемости процессов

### 3. Метрики оценки

| Метрика | Формула | Описание | Пример |
|---------|---------|----------|--------|
| **ROI** | (Выгоды − Затраты) / Затраты × 100% | Рентабельность инвестиций | ROI = 150% |
| **Payback Period** | Затраты / Ежемесячная выгода | Срок окупаемости | 8 месяцев |
| **NPV** | Σ(Выгоды_года / (1+r)^год) − Затраты | Чистая приведённая стоимость | $120K > 0 |
| **TCO** | Сумма всех затрат за период | Total Cost of Ownership | $50K за 3 года |

## Пример CBA

**Задача:** Автоматизировать согласование договоров (заменить email → система workflow).

### Costs

| Статья | Сумма |
|--------|-------|
| Разработка (4 месяца × 2 разработчика) | 40,000 |
| Интеграция с 1С и ЭДО | 15,000 |
| Обучение сотрудников | 5,000 |
| Лицензия (год) | 12,000 |
| **Total Costs (Year 1)** | **72,000** |

### Benefits (ежегодно)

| Статья | Сумма |
|--------|-------|
| Экономия времени юристов (2 часа × $50 × 200 дней) | 20,000 |
| Ускорение сделок (меньше просрочек контрактов, +5% выручки) | 50,000 |
| Снижение штрафов за просрочку | 10,000 |
| **Total Benefits (год)** | **80,000** |

### Результат

- **ROI:** (80,000 - 72,000) / 72,000 = **11%** (первый год)
- **Payback:** 72,000 / 80,000 ≈ **10.8 месяцев**
- **Вывод:** проект окупается менее чем за год, запускаем

## Когда CBA необходим

- Выбор между «сделать самим» и «купить готовое»
- Оценка MVP vs полного функционала
- Бюджетное обоснование перед стейкхолдерами
- Сравнение технических альтернатив (например, AWS vs on-premise)

## Когда CBA не работает

- **Нет данных** — оценки «из головы» хуже, чем отсутствие CBA
- **Качественные выгоды сложно измерить** — «улучшение UX» не конвертируется в $ без A/B-теста
- **Политическое решение** — CTO уже выбрал решение, CBA нужен «для галочки»

## Как улучшить точность CBA

- Используйте **три сценария**: оптимистичный / реалистичный / пессимистичный
- Добавьте **дисконтирование** — $ сегодня дороже $ через год
- Учитывайте **неопределённость** — указывайте диапазон (P50, P90)
- Пересчитывайте **после внедрения** — факт vs план
