:::info TL;DR
OMS (Order Management System) — система управления заказами, ядро e-commerce платформы. Заказ проходит через десяток статусов: от «создан» до «доставлен» или «возвращён». Аналитик должен специфицировать машину состояний заказа, интеграции с платежами, складами и доставкой.
:::

## Жизненный цикл заказа

```
Создан ──→ Ожидает оплаты ──→ Оплачен ──→ Собран ──→ Передан в доставку
  │                              │                      │
  │                              ▼                      ▼
  └──→ Отменён            ──→ Возврат              Доставлен
                                                      │
                                                      ▼
                                                  Получен
```

### Ключевые статусы заказа

| Статус | Что происходит | Система-владелец |
|--------|---------------|-----------------|
| `CREATED` | Заказ создан, товар зарезервирован | OMS |
| `PENDING_PAYMENT` | Ожидание оплаты (20 мин — 24ч) | Платёжный шлюз |
| `PAID` | Оплачен полностью | OMS |
| `CONFIRMED` | Подтверждён менеджером/автоматом | OMS |
| `ASSEMBLING` | Собирается на складе | WMS |
| `ASSEMBLED` | Собран, ожидает передачи в доставку | WMS |
| `SHIPPED` | Передан курьеру/ПВЗ | TMS |
| `DELIVERED` | Доставлен в ПВЗ | TMS |
| `RECEIVED` | Получен клиентом | OMS |
| `CANCELLED` | Отменён | OMS |
| `RETURNED` | Возвращён | OMS |
| `REFUNDED` | Деньги возвращены | Платёжный шлюз |

## Состав заказа (сущности)

```
Order
 ├── id, created_at, status, total
 ├── Customer (id, name, phone, email)
 ├── BillingAddress
 ├── ShippingAddress
 ├── Items[]
 │    ├── product_id, sku, name, quantity, price
 │    └── reservation (stock_id, status)
 ├── Payments[]
 │    └── amount, method, status, external_id
 ├── Shipments[]
 │    └── carrier, tracking_number, status
 └── History[]
      └── status, timestamp, user, comment
```

## Резервирование товара (Inventory Reservation)

Ключевая проблема — товар может закончиться, пока клиент оплачивает.

```
1. Клиент положил товар в корзину → бронь на 20 мин (HELD)
2. Клиент оформил заказ → бронь подтверждена (CONFIRMED)
3. Оплата не прошла → бронь снята (RELEASED)
4. Оплата прошла → бронь → реальное списание со склада (PICKED)
```

**Стратегии резервирования:**
- **Optimistic** — не резервируем до оплаты (риск oversell)
- **Pessimistic** — резервируем при добавлении в корзину (риск брошенных корзин)
- **Hybrid** — резервируем на N минут, затем отпускаем

## Интеграции OMS

OMS — интеграционный хаб e-commerce платформы:

```
PMS (Платежи) ←──→ OMS ←──→ WMS (Склад)
                        ↑
                   TMS (Доставка)
                        ↑
                   ERP (Учёт)
                        ↑
                   Маркетплейсы
```

### Типовые интеграции

| Система | Что передаём | Протокол |
|---------|-------------|----------|
| WMS | Товары к сборке, статус сборки | REST / MQ |
| TMS | Заказ на доставку, трекинг | REST API |
| ERP | Проводки, себестоимость | REST / EDI / 1С |
| Платёжный шлюз | Сумма к оплате, статус | REST |
| Маркетплейс | Заказы из маркетплейса | REST API |

## Гранулярность товаров в OMS

| Понятие | Описание | Пример |
|---------|----------|--------|
| **SKU** | Уникальный ID товара | 12345-IPHONE-15-BLK |
| **EAN** | Штрихкод (международный) | 4601234567890 |
| **Lot** | Партия товара | L2024-123 |
| **Serial** | Серийный номер (для гарантии) | SN-A1B2C3 |
| **Bundle** | Набор товаров | Конструктор LEGO + коробка |
| **Kit** | Комплект со скидкой | Кроссовки + носки — 10% |

## Требования к OMS (спецификация)

| Параметр | Пример |
|----------|--------|
| Поддерживаемые статусы | 12+ статусов (см. выше) |
| Резервирование | Hybrid (20 мин на корзину) |
| Источники товара | Собственный склад + FBO + FBS |
| Доставка | 5+ служб (CDEK, Boxberry, Почта) |
| Оплата | Карты, SBP, BNPL, наличные |
| Возвраты | 14 дней (ЗоЗПП), 30 дней (онлайн) |
| SLA | Заказ → отправка: 24 часа |
| Пиковая нагрузка | 10 000 заказов/час (Black Friday) |

## Проверь себя

1. **Какие статусы проходит заказ от создания до получения?**
   *Ответ:* CREATED → PENDING_PAYMENT → PAID → CONFIRMED → ASSEMBLING → ASSEMBLED → SHIPPED → DELIVERED → RECEIVED.

2. **Что такое резервирование товара и какие есть стратегии?**
   *Ответ:* Бронь товара для клиента. Optimistic (не резервируем), Pessimistic (резервируем в корзине), Hybrid (на N минут).

3. **С какими системами интегрируется OMS?**
   *Ответ:* WMS (склад), TMS (доставка), ERP (учёт), платёжный шлюз, маркетплейсы.
