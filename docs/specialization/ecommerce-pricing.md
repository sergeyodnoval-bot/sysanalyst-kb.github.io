:::info TL;DR
Ценообразование в e-commerce — сложная система с перекрывающимися скидками, промокодами, программами лояльности и динамическими ценами. Аналитик должен специфицировать правила расчёта финальной цены, приоритеты скидок и избегать «схлопывания» (когда скидки делают цену отрицательной).
:::

## Типы цен

| Тип | Описание | Пример |
|-----|----------|--------|
| **Base price** | Базовая цена товара (в каталоге) | 9 990 ₽ |
| **Sale price** | Цена по акции (скидка от base) | 7 990 ₽ |
| **Promo price** | Цена по временной акции | 6 990 ₽ |
| **Wholesale price** | Оптовая цена (B2B) | 5 990 ₽ |
| **Subscription price** | Цена по подписке | 499 ₽/мес |
| **Dynamic price** | Меняется в зависимости от спроса | Авиабилеты, Uber |

## Система скидок

### Каналы скидок

```
┌─────────────────────────────────────────┐
│            Финальная цена                │
├─────────────────────────────────────────┤
│ Base price                              │
│   - Скидка по промокоду                 │
│   - Скидка по программе лояльности      │
│   - Скидка за количество (volume)       │
│   - Скидка за способ оплаты             │
│   - Скидка за самовывоз                 │
│   + Наценка за срочность                │
│   + Наценка за регион                   │
├─────────────────────────────────────────┤
│ = Price to pay                          │
└─────────────────────────────────────────┘
```

### Виды промо-акций

| Тип | Механика | Пример |
|-----|----------|--------|
| **Flash sale** | Скидка N% на N часов | -30% на электронику, 12:00–15:00 |
| **Bundle** | Набор со скидкой | Ноутбук + мышь = -15% |
| **Volume** | Скидка от количества | 2 по цене 1 |
| **Cashback** | Возврат баллами | 10% кешбэк на следующую покупку |
| **Gift** | Подарок при покупке | Шампунь в подарок при заказе от 3 000 ₽ |
| **Бесплатная доставка** | Доставка 0 ₽ от N ₽ | Бесплатно от 2 500 ₽ |

## Промокоды (Coupon Engine)

```
Промокод
 ├── code: "SALE30"
 ├── type: fixed / percent / free_delivery
 ├── value: 30 (%) / 500 (₽)
 ├── conditions:
 │    ├── min_order: 2 000 ₽
 │    ├── max_discount: 5 000 ₽
 │    ├── categories: [electronics, books]
 │    ├── products: [IPHONE-15]
 │    ├── first_order_only: true
 │    └── valid_period: 2024-11-01..2024-11-30
 ├── usage:
 │    ├── max_uses: 1 000
 │    ├── per_user: 1
 │    └── used_count: 327
 └── stackable: false (не суммируется с другими)
```

## Приоритет скидок (Stacking Rules)

**Проблема:** если применить все скидки сразу, цена может стать < 1 ₽.

**Правило приоритета (пример):**

```
1. Base price
2. Скидка по промокоду (если не stackable → stop)
3. Скидка по лояльности (суммируется с % от base)
4. Volume discount
5. Бесплатная доставка (не влияет на цену товара)
6. Cashback (не влияет на цену сейчас, начисляется потом)
```

**Stackable vs Non-stackable:**
- **Stackable** — суммируется с другими скидками (например, лояльность + промокод)
- **Non-stackable** — заменяет другие скидки (выбирается максимальная)

## Динамическое ценообразование (Dynamic Pricing)

Цена меняется в зависимости от:

| Фактор | Пример |
|--------|--------|
| Спрос | Больше просмотров → цена выше |
| Остатки | Осталось 3 шт → цена выше |
| Конкуренты | У конкурента дешевле → цена ниже |
| Время дня | Ночью скидка на доставку |
| Сегмент клиента | VIP → своя цена |
| Гео | Разная цена для разных регионов |

## Требования к системе ценообразования

| Параметр | Пример |
|----------|--------|
| Типы скидок | percent, fixed, gift, delivery |
| Условия промокода | min_order, max_discount, категории, товары |
| Stacking | Да (с приоритетами) |
| Максимальная скидка | 99% (не может быть 100%+) |
| Периоды акций | С точностью до минуты |
| Аудит | Кто, когда и почему изменил цену |
| Тестирование | A/B тест для новой схемы скидок |

## Проверь себя

1. **Какие есть типы цен в e-commerce?**
   *Ответ:* Base, sale, promo, wholesale, subscription, dynamic.

2. **Что такое stacking правил и почему это важно?**
   *Ответ:* Приоритет применения скидок. Без него несколько скидок могут сделать цену отрицательной.

3. **Какие условия можно задать для промокода?**
   *Ответ:* min_order, max_discount, категории, конкретные товары, только первый заказ, период действия, лимит использований.
