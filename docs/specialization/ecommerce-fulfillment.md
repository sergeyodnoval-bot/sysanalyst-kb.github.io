:::info TL;DR
Фулфилмент — это «последняя миля» e-commerce: сборка заказа на складе, упаковка, передача в доставку и трекинг до клиента. Состоит из этапов Pick → Pack → Ship. Аналитик специфицирует интеграцию с WMS, логику распределения заказов по складам и работу с возвратами.
:::

## Процесс фулфилмента (Pick → Pack → Ship)

```
Заказ в OMS (статус ASSEMBLING)
    │
    ▼
Pick (Сборка)
    │
    ├── Сборщик получает задание (терминал)
    ├── Маршрут по складу (оптимизация)
    ├── Сканирование товаров штрихкодом
    └── Проверка комплектности
    │
    ▼
Pack (Упаковка)
    │
    ├── Выбор коробки/пакета по размеру
    ├── Добавление накладной, чеков
    ├── Качественный контроль
    └── Взвешивание
    │
    ▼
Ship (Отгрузка)
    │
    ├── Сортировка по службам доставки
    ├── Создание заказа в TMS
    ├── Печать наклеек (tracking number)
    └── Передача курьеру/в ПВЗ
```

## Типы складов (WMS)

| Тип | Описание | Для каких моделей |
|-----|----------|-------------------|
| **Собственный склад** | Компания арендует/владеет складом | 1P, FBO |
| **Fulfillment by Operator** | Маркетплейс хранит товары продавца | FBO |
| **Склад продавца** | Товар физически у продавца | FBS, DBS |
| **Cross-docking** | Транзитный склад (товар не хранится) | Свежие продукты |
| **Dark store** | Склад-магазин (как ПВЗ, только для сборки) | Быстрая доставка |

## Зоны склада

```
Приёмка → Хранение → Pick → Pack → Sort → Ship
   │                                    │
   ▼                                    ▼
  Quality Control (QC)           Returns (RMA)
                                      │
                                      ▼
                           Inspection → Склад возврата / Утилизация
```

## Интеграция OMS → WMS

OMS не хранит остатки, он запрашивает их у WMS:

```
Запрос:     OMS → WMS: «Зарезервируй 2 единицы товара 12345»
Ответ:      WMS → OMS: «Зарезервировано, ID резерва R-98765»
Сборка:     WMS → OMS: «Товар собран, заказ 45678 в статусе PACKED»
Отгрузка:   WMS → OMS: «Заказ 45678 передан CDEK, трек CDEK123456»

Тип синхронизации: асинхронная (RabbitMQ / Kafka)
```

## Службы доставки

| Служба | Тип | Особенности |
|--------|-----|-------------|
| **CDEK** | Курьер + ПВЗ | Постаматы, международная |
| **Boxberry** | ПВЗ | Широкая сеть в регионах |
| **Почта России** | Почта | Долгая, но дешёвая |
| **Яндекс.Доставка** | Курьер | Быстрая, только города |
| **DHL / Pony Express** | Курьер | Дорогая, для B2B |

**Для аналитика:** каждая служба — отдельная интеграция (REST API), свой формат накладных, SLA и зоны покрытия.

## Возвраты (RMA)

**Return Merchandise Authorization — процесс возврата:**

```
1. Клиент создаёт запрос на возврат
2. OMS проверяет условия (срок, состояние)
3. Создаётся RMA-запрос
4. Клиент передаёт товар (курьер / ПВЗ)
5. Приёмка → Inspection (QC)
6. Решение: принят / отклонён
7. Если принят → возврат денег + товар на склад
8. Если отклонён → товар назад клиенту
```

**Виды возвратов:**
- **Carrier return** — клиент отказывается на доставке
- **Quality return** — брак / повреждение
- **Change of mind** — 14 дней (ЗоЗПП)
- **Warranty return** — гарантийный случай

## Требования к фулфилменту (спецификация)

| Параметр | Пример |
|----------|--------|
| SLA сборки | < 4 часа с момента заказа |
| SLA отгрузки | < 24 часа до передачи в TMS |
| Склады | 3 (Москва, СПб, Екатеринбург) |
| Службы доставки | CDEK, Почта, Яндекс (API) |
| Возвраты | 14 дней (ЗоЗПП), 30 (онлайн) |
| QC | 5% выборочная проверка |
| Ошибка сборки | < 0.5% заказов |

## Проверь себя

1. **Какие этапы проходит заказ в фулфилменте?**
   *Ответ:* Pick (сборка) → Pack (упаковка) → Ship (отгрузка). Перед этим — приёмка на склад.

2. **Как OMS взаимодействует с WMS?**
   *Ответ:* Асинхронно через MQ: OMS → WMS (резервирование), WMS → OMS (статусы сборки и отгрузки).

3. **Что такое RMA и какие есть виды возвратов?**
   *Ответ:* Return Merchandise Authorization — процесс возврата. Виды: отказ на доставке, брак, change of mind, гарантия.
