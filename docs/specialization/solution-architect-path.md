# Путь Solution-архитектора

Solution Architect (SArch) — это роль, которая соединяет **бизнес и техническую реализацию**. Solution Architect проектирует решение на уровне системы: выбор технологий, интеграции, архитектурные стили, оценка стоимости.

## Чем занимается Solution Architect

- **Проектирование решения** — выбор архитектуры, компонентов, технологий
- **Оценка** — TCO, сроки, риски, альтернативы (proof-of-concept)
- **Спецификация** — ADR, архитектурные диаграммы (C4), API-контракты
- **Коммуникация** — между бизнесом и командами разработки
- **Стандарты** — code review архитектурных решений, tech radar

## Компетенции

| Область | Что нужно знать |
|---------|----------------|
| **Архитектура** | DDD, Event-Driven, CQRS, Layered, Hexagonal, микросервисы |
| **Интеграции** | REST, gRPC, Kafka, RabbitMQ, API Gateway, EIP |
| **Инфраструктура** | Docker, Kubernetes, облака (AWS / Azure / Yandex Cloud) |
| **Безопасность** | OAuth2, JWT, TLS, SOC2 (базовый уровень) |
| **Базы данных** | SQL / NoSQL, выбор БД под задачу, миграции |
| **Soft skills** | Презентация C-level, переговоры, менторство |

## SA vs SA (Scrum Master vs Architect)

Важно не путать:
- **SA (System Analyst)** — требования, спецификации, детальное проектирование
- **SA (Solution Architect)** — архитектура, технологии, решение целиком

## Карьерный путь

```
System Analyst → Senior SA → Solution Architect / Enterprise Architect
                                    ↓
                         Enterprise Architect (стратегия)
                         Technical Lead / CTO (в стартапе)
```
