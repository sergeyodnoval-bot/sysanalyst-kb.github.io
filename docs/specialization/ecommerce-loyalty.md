:::info TL;DR
Программа лояльности — система поощрения повторных покупок: баллы, кешбэк, уровни (tiers), эксклюзивные предложения. Аналитик специфицирует правила начисления и списания баллов, условия получения уровня, интеграцию с OMS и синхронизацию баланса в реальном времени.
:::

## Типы программ лояльности

| Тип | Механика | Пример |
|-----|----------|--------|
| **Балльная** | N баллов за каждую покупку | Спасибо от Сбера |
| **Кешбэк** | N% от суммы покупки | 10% баллами |
| **Уровневая (Tier)** | Платиновый, Золотой, Серебряный | Аэрофлот Бонус |
| **Подписочная** | Платный доступ к привилегиям | Яндекс.Плюс |
| **Реферальная** | Бонус за приведённого друга | Airbnb |
| **Геймификация** | Достижения, миссии, челленджи | Starbucks Rewards |

## Как работает балльная система

```
Начисление:
  Покупка: 1 000 ₽
  Rate: 5% баллами
  Начислено: 50 баллов

Списание:
  Товар: 2 000 ₽
  Можно списать: до 30% (600 ₽ баллами)
  Курс: 1 балл = 1 ₽
  Списано: 500 баллов
  К оплате: 1 500 ₽

Истечение:
  Баллы сгорают через 365 дней без активности
```

## Уровневая система (Tiers)

| Уровень | Условие | Привилегии |
|---------|---------|------------|
| **Silver** | Регистрация | Базовая, 1 балл = 1 ₽ |
| **Gold** | > 10 000 ₽ за 6 мес | 1.5 балла = 1 ₽ + бесплатная доставка |
| **Platinum** | > 50 000 ₽ за 6 мес | 2 балла = 1 ₽ + приоритетная поддержка |
| **VIP** | > 200 000 ₽ за 6 мес | 3 балла = 1 ₽ + персональный менеджер |

**Период действия уровня:** 6 месяцев, затем пересчёт. Если за 6 мес не набрал нужную сумму — понижение на уровень.

## Events в системе лояльности

```
order.created        → начислить баллы (по rate уровня)
order.refunded       → списать начисленные баллы
loyalty.burn         → списать баллы при покупке
loyalty.expire       → сгорание баллов (cron)
loyalty.tier.up      → повышен уровень
loyalty.tier.down    → понижен уровень
referral.invited     → бонус за приглашение
referral.purchased   → бонус за покупку друга
```

## Интеграция с OMS

```
1. OMS: заказ оплачен (status = PAID)
2. OMS → Loyalty: event order.paid (customer_id, total)
3. Loyalty: начисляет баллы (rate = 5% × total)
4. Loyalty: проверяет уровень (total за 6 мес)
5. Loyalty: проверяет активацию (не истекли ли баллы)
6. Loyalty: обновляет баланс клиента
7. Клиент видит новый баланс в личном кабинете
```

**Важно:** начисление должно происходить после подтверждения оплаты, а не после оформления заказа (иначе можно начислить баллы на отменённый заказ).

## Требования к системе лояльности (спецификация)

| Параметр | Пример |
|----------|--------|
| Типы | Баллы + уровни + рефералка |
| Rate начисления | 5% (Silver), 10% (Gold), 15% (Platinum) |
| Курс балла | 1 балл = 1 ₽ (фиксированный) |
| Макс. списание | 50% от суммы заказа |
| Срок баллов | 365 дней |
| Уровни | 4 (Silver, Gold, Platinum, VIP) |
| Период пересчёта | 6 месяцев |
| Реферальный бонус | 500 баллов при первой покупке друга |
| Аудит | Все начисления/списания логируются |

## Проверь себя

1. **Какие есть типы программ лояльности?**
   *Ответ:* Балльная, кешбэк, уровневая (tiers), подписочная, реферальная, геймификация.

2. **Как работает начисление и списание баллов?**
   *Ответ:* Начисление — N% от суммы покупки (после оплаты). Списание — по курсу 1 балл = X ₽, с ограничением (до 30–50% от суммы). Баллы могут сгорать.

3. **Когда нужно начислять баллы — после оформления или после оплаты?**
   *Ответ:* Только после подтверждения оплаты (иначе баллы начислятся на отменённый заказ).
