:::info TL;DR
Каталог товаров — центральная сущность e-commerce. Структура (категории, атрибуты, SKU), поиск (Elasticsearch), инвентаризация (остатки, резервы) и ценообразование. Типичная ошибка новичка — проектировать каталог как плоскую таблицу.
:::

## Структура каталога

Товар в e-commerce имеет иерархию:

```
Category (Категория)
 ├── Subcategory (Подкатегория)
 │    └── Product Model (Модель товара)
 │         ├── SKU-1 (iPhone 15 Black 128GB)
 │         ├── SKU-2 (iPhone 15 White 128GB)
 │         └── SKU-3 (iPhone 15 Black 256GB)
 │              ├── price, weight, barcode
 │              ├── stock: { warehouse_1: 10, warehouse_2: 5 }
 │              └── attributes: { color, memory, storage }
```

| Понятие | Описание | Пример |
|---------|----------|--------|
| **Category** | Категория товара | Электроника → Смартфоны |
| **Product** | Абстрактный товар | iPhone 15 |
| **Variant** | Вариант (SKU) | iPhone 15 Black 128GB |
| **SKU** | Stock Keeping Unit | IP15-BLK-128 (код) |
| **EAN/UPC** | Штрихкод | 4601234567890 |
| **Attribute** | Характеристика | Цвет, память, размер |

## Атрибуты товара

Атрибуты делятся на:

| Тип | Пример | Использование |
|-----|--------|-------------|
| **Фиксированные** | Вес, бренд, страна | Фильтры, карточка товара |
| **Вариативные** | Цвет, размер, память | Выбор варианта (SKU) |
| **Технические** | Процессор, экран | Карточка товара |
| **SEO** | Title, description, keywords | Поиск, мета-теги |

**Для аналитика:** специфицировать, какие атрибуты влияют на SKU (вариативные), какие просто описывают товар.

## Поиск по каталогу (Elasticsearch)

Современный каталог использует Elasticsearch для быстрого поиска:

```
Пользователь → "iPhone 15"
    │
    ▼
Elasticsearch
    ├── full-text search (название, описание)
    ├── фильтры (цена, бренд, цвет)
    ├── аггрегации (сколько товаров в каждой категории)
    ├── сортировка (по цене, популярности)
    └── релевантность (boost по бренду, рейтингу)
    │
    ▼
Результаты (product_id → SKU → остатки → цена)
```

**Требования к поиску:**
- **Fuzzy search** — «айфон» найдёт «iPhone»
- **Autocomplete** — подсказки при вводе
- **Synonym dictionary** — «мобила» = «смартфон»
- **Faceted search** — фильтры с количеством товаров
- **Геопоиск** — товары рядом с ПВЗ

## Инвентаризация (Inventory Management)

Учёт остатков товара:

| Термин | Описание |
|--------|----------|
| **Stock** | Физический остаток на складе |
| **Reserved** | Зарезервировано для заказов в обработке |
| **Available** | Доступно к продаже = Stock - Reserved |
| **In transit** | В пути между складами |
| **Defective** | Брак |
| **Quarantine** | На проверке (возврат, брак) |

### Методы учёта

| Метод | Описание | Когда используют |
|-------|----------|----------------|
| **FIFO** | First In First Out — товар списывается с первой партии | Продукты, косметика |
| **LIFO** | Last In First Out — товар списывается с последней партии | Стройматериалы (редко) |
| **FEFO** | First Expiry First Out — сначала ближайший срок годности | Лекарства |
| **Average cost** | Средняя себестоимость | Одежда, электроника |
| **Lot tracking** | По партиям | Гарантия, отзыв товара |

## Остатки в реальном времени

**Проблема:** каталог (Elasticsearch) и склад (WMS) — разные системы.

```
Схема согласования:
Заказ → OMS → WMS (резервирование)
                    ↓
WMS → OMS → Каталог (обновление остатка)
                    ↓
         Elasticsearch (перестроение индекса)
```

Задержка между списанием на складе и обновлением в каталоге может быть 1–5 минут. Для пиковых распродаж это критично → oversell.

**Решения:**
- **Pre-reserve** — держать в каталоге запас (safety stock) на время лага
- **Sync via MQ** — немедленное уведомление через Kafka/RabbitMQ
- **Throttling** — ограничить количество товаров в корзине на время распродажи

## Проверь себя

1. **Чем отличается Product Model от SKU?**
   *Ответ:* Product Model — абстрактный товар (iPhone 15). SKU — конкретный вариант к продаже (iPhone 15 Black 128GB).

2. **Как работает поиск в каталоге e-commerce?**
   *Ответ:* Elasticsearch: full-text + фильтры + аггрегации + релевантность. Данные приходят из БД каталога через пайплайн синхронизации.

3. **Какие есть методы учёта товара на складе?**
   *Ответ:* FIFO (первый пришёл — первый ушёл), LIFO, FEFO (по сроку годности), average cost, lot tracking.
