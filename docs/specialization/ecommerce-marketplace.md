:::info TL;DR
Маркетплейс — платформа, где продавцы (sellers) продают товары покупателям, а маркетплейс берёт комиссию. Ключевые модели: FBO (маркетплейс хранит товар), FBS (продавец хранит сам), DBS (продавец и доставляет сам). Аналитик проектирует онбординг продавцов, комиссионную модель, интеграцию с системами продавцов.
:::

## Модели маркетплейсов

| Модель | Кто хранит | Кто доставляет | Пример |
|--------|-----------|---------------|--------|
| **1P (First Party)** | Маркетплейс | Маркетплейс | Яндекс.Маркет (свои товары) |
| **3P (Third Party)** | Продавец | Продавец | Etsy, eBay |
| **Hybrid** | И то, и другое | И то, и другое | Ozon, Wildberries |
| **Dropshipping** | Поставщик | Поставщик | AliExpress |

### FBO vs FBS vs DBS

| Параметр | FBO (Full By Operator) | FBS (Full By Seller) | DBS (Delivery By Seller) |
|----------|----------------------|---------------------|--------------------------|
| Хранение | Склад маркетплейса | Склад продавца | Склад продавца |
| Доставка | Маркетплейс | Логистика маркетплейса | Продавец |
| Срок доставки | 1–2 дня | 2–5 дней | 3–10 дней |
| Комиссия | Высокая | Средняя | Низкая |
| Возврат | Маркетплейс | Маркетплейс → продавец | Продавец |

## Комиссионная модель

Маркетплейс зарабатывает на комиссии с каждой продажи:

```
Цена товара: 1 000 ₽
Комиссия (15%): -150 ₽
Логистика: -50 ₽
Эквайринг: -20 ₽
───────────────
Продавец получит: 780 ₽
Маркетплейс заработал: 220 ₽
```

**Факторы комиссии:**
- Категория товара (электроника — низкая, одежда — высокая)
- Модель (FBO — выше комиссия, DBS — ниже)
- Объём продаж (объёмные скидки на комиссию)
- Рейтинг продавца

## Онбординг продавца (Seller Onboarding)

```
1. Регистрация (email, телефон)
2. Верификация (паспорт, ИНН, ОГРНИП/ОГРН)
3. Подписание оферты (УКЭП)
4. Добавление товаров (API / XLS / CSV)
5. Настройка цен и остатков (API)
6. Выбор модели (FBO / FBS / DBS)
7. Тестовый заказ
8. Запуск
```

## Интеграция с продавцами (Seller API)

Продавец подключается через REST API:

```
POST   /api/seller/products              — загрузить товары
PUT    /api/seller/products/{sku}/stock   — обновить остатки
PUT    /api/seller/products/{sku}/price   — обновить цену
GET    /api/seller/orders                 — список заказов
POST   /api/seller/orders/{id}/confirm    — подтвердить заказ
POST   /api/seller/orders/{id}/ship       — отгрузить заказ
GET    /api/seller/reports/sales          — отчёт по продажам
GET    /api/seller/reports/commission     — отчёт по комиссиям
```

## Статусы заказа в маркетплейсе

```
NEW ──→ CONFIRMED ──→ PICKING ──→ SHIPPED ──→ DELIVERED ──→ COMPLETED
 │          │            │                           │
 ▼          ▼            ▼                           ▼
CANCELLED   RETURN_INIT  LOST                     RETURNED
    (до подтверждения)   (утерян)                  (возврат)
```

## Карточка товара на маркетплейсе

Маркетплейс решает, какой товар показать первым (ранжирование):

| Фактор | Влияние |
|--------|---------|
| Цена + доставка | Чем ниже, тем выше |
| Рейтинг продавца | 1–5 звёзд |
| Оборачиваемость | Больше продаж = выше |
| Возвраты | Меньше возвратов = выше |
| Контент (фото, описание) | Полнота карточки |
| Рекламный бюджет | Доплата за показы |

## Проверь себя

1. **Чем FBO отличается от FBS и DBS?**
   *Ответ:* FBO — товар на складе маркетплейса, доставка маркетплейсом (дорого, быстро). FBS — товар у продавца, доставка маркетплейсом. DBS — всё у продавца (дёшево, медленно).

2. **Из чего складывается комиссия маркетплейса?**
   *Ответ:* Процент от продажи + логистика + эквайринг. Зависит от категории, модели (FBO/FBS) и объёма.

3. **Какие статусы проходит заказ в маркетплейсе?**
   *Ответ:* NEW → CONFIRMED → PICKING → SHIPPED → DELIVERED → COMPLETED, плюс CANCELLED / RETURNED / LOST.
