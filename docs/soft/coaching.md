# Коучинг — базовые техники

Коучинг — это метод развития, при котором коуч **не даёт советов**, а задаёт вопросы, помогающие человеку самому найти решение.

## Когда нужен коучинг, а когда совет

| Ситуация | Что нужно |
|----------|-----------|
| Менти **не знает**, как сделать | Менторство (совет, пример) |
| Менти **знает**, но не уверен | Коучинг (вопросы) |
| Менти **может**, но не делает | Коучинг (мотивация) |
| Менти **хочет** расти, но не знает куда | Коучинг (цели) |
| Кризис, ошибка, проблема | Сначала коучинг, потом менторство |

## Основные техники коучинга

### 1. Powerful Questions (Сильные вопросы)

Вопросы, которые нельзя ответить «да/нет»:

| Вместо | Спросите |
|--------|----------|
| «Ты пробовал BPMN?» | «Какие способы моделирования ты рассматривал?» |
| «Почему ты не сделал?» | «Что тебе помешало сделать?» |
| «Это хорошее решение?» | «По каким критериям ты оцениваешь решение?» |

### 2. GROW Model

| Этап | Вопросы |
|------|---------|
| **G**oal | «Что ты хочешь достичь? Как узнаешь, что достиг?» |
| **R**eality | «Где ты сейчас? Что уже пробовал?» |
| **O**ptions | «Какие варианты есть? Что ещё можно сделать?» |
| **W**ill | «Что выберешь? Когда начнёшь? Как я могу помочь?» |

### 3. Active Listening

- Не перебивать
- Перефразировать: «Правильно ли я понял, что…»
- Резюмировать: «Итак, основные проблемы…»
- Паузы: молчание даёт пространство для размышления

### 4. Scaling (Шкалирование)

```
«Оцени по шкале от 1 до 10, насколько ты уверен в этом решении?»
  → «4» 
  → «Что нужно, чтобы было 7?»
  → «Нужны данные по latency альтернативного варианта»
  → «Где их взять?»
```

### 5. Reframing (Переформулирование)

Помогает увидеть ситуацию с другой стороны.

```
Аналитик: «Я не могу договориться с заказчиком, он всё время меняет требования»

Рефрейминг: «Как ты думаешь, может, он не до конца понимает, что ему нужно,
             и ищет правильное решение? Как помочь ему определиться?»
```

## Коучинг vs Терапия

| | Коучинг | Терапия |
|--|---------|---------|
| **Фокус** | Настоящее → будущее | Прошлое → настоящее |
| **Проблема** | Рабочая: навыки, решения | Личная: тревога, травмы |
| **Кто** | Здоровый человек | Человек с псих. проблемой |
| **Цель** | Достичь цели | Исцелить травму |

**Аналитик не должен заниматься терапией коллег.** Если видите глубокие личные проблемы — рекомендовать психолога.

## Как внедрить коучинг в работу

- **1:1 с менти** — первые 10 минут коучинг (как дела? что волнует?)
- **Ретроспектива** — вместо «надо исправить» → «что мы можем сделать иначе?»
- **Сбор требований** — вместо «скажите, что нужно» → «как вы видите идеальный процесс?»
- **Разбор ошибки** — вместо «вот что ты сделал не так» → «что бы ты сделал иначе в следующий раз?»

## Когда НЕ использовать коучинг

- **Кризис** — пожар в проде → нужны чёткие инструкции
- **Нет компетенции** — человек не знает SQL → менторство, а не коучинг
- **Нет времени** — 5 минут до дедлайна → дайте ответ

## Типичные ошибки

- **Путать с менторством** — коучинг не про советы, а про вопросы
- **Слишком много вопросов** — похоже на допрос
- **Нет структуры** — вопросы хаотичные, человек теряется
- **Навязывать ответ** — «тебе нужно сделать X, правда?» (это не коучинг)
