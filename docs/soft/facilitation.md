# Фасилитация встреч

Фасилитация — это управление процессом группового обсуждения для достижения **конкретного результата**. Фасилитатор не участник, он модерирует процесс, чтобы группа пришла к решению.

## Роль фасилитатора

| Что делает | Что НЕ делает |
|-----------|---------------|
| Управляет повесткой | Принимает решения за группу |
| Следит за таймингом | Доминирует в обсуждении |
| Вовлекает всех участников | Позволяет одному говорить за всех |
| Фиксирует результаты | Уходит в детали одной темы |
| Снимает конфликты | Игнорирует напряжённость |

## Когда нужна фасилитация

- **Сбор требований** — группа стейкхолдеров не может договориться
- **Ретроспектива** — нужно, чтобы команда сама нашла улучшения
- **Event Storming** — моделирование процессов с большой группой
- **Приоритизация** — бизнес и разработка не сходятся в приоритетах
- **Интервью** — не структурированное, а групповое

## Структура встречи

### Подготовка

1. **Цель** — что должно быть на выходе? (решение, список, диаграмма)
2. **Повестка** — тайминг на каждую тему
3. **Участники** — кто нужен для решения?
4. **Материалы** — стикеры, флипчарт, Miro

### Проведение

```
  ┌────────────────────────────────────────────┐
  │  1. Opening (5 мин)                        │
  │  Цель, повестка, правила                   │
  ├────────────────────────────────────────────┤
  │  2. Divergent thinking (15–30 мин)         │
  │  Генерация идей, сбор мнений               │
  ├────────────────────────────────────────────┤
  │  3. Convergent thinking (15–30 мин)        │
  │  Группировка, приоритизация, голосование   │
  ├────────────────────────────────────────────┤
  │  4. Decision (10–15 мин)                   │
  │  Фиксация решения, next steps              │
  ├────────────────────────────────────────────┤
  │  5. Closing (5 мин)                        │
  │  Ретроспектива встречи                     │
  └────────────────────────────────────────────┘
```

### После встречи

- Протокол (решения, action items, кто ответственный, дедлайн)
- Опубликовать в Confluence / чате команды

## Техники фасилитации

| Техника | Когда применять |
|---------|----------------|
| **Round-robin** | Кто-то доминирует в обсуждении |
| **Silent brainstorming** | Нужны идеи без влияния группы |
| **Dot Voting** | Быстрая приоритизация |
| **World Cafe** | Большая группа, несколько тем |
| **Five Whys** | Поиск корневой причины проблемы |
| **Plus/Delta** | Ретроспектива (что хорошо / что менять) |
| **Elevator pitch** | Сформулировать суть за 30 секунд |

### Dot Voting

Каждый участник получает N стикеров-точек и клеит их напротив самых важных пунктов:

```
  [x] Внедрить онлайн-оплату       ●●●●●●
  [x] Чат с поддержкой              ●●●
  [x] История заказов               ●●●●●
  [x] Промокоды                     ●
```

## Трудные ситуации

| Ситуация | Что делать |
|----------|-----------|
| **Один говорит за всех** | «Спасибо, Иван. А что думают остальные?» (round-robin) |
| **Спор перешёл в конфликт** | «Давайте зафиксируем оба мнения и вернёмся к ним через 5 минут» |
| **Ушли в детали** | «Это важный вопрос, давайте вынесем его в отдельную встречу» |
| **Тишина** | «Давайте каждый напишет на стикере по 3 идеи» (silent brainstorm) |
| **Опоздания** | Не ждать больше 5 минут, начать вовремя |

## Типичные ошибки фасилитатора

- **Фасилитатор = участник** — нельзя одновременно вести встречу и участвовать
- **Нет повестки** — встреча превращается в «болтовню»
- **Нет тайминга** — обсуждение первого пункта заняло час, остальные не успели
- **Нет результата** — поговорили и разошлись без решений
- **Игнор тихих участников** — самые ценные идеи часто у молчаливых
