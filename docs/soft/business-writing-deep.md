# Деловая переписка и коммуникация

Аналитик много пишет: письма, протоколы, спецификации, сообщения в чатах. Качество письменной коммуникации напрямую влияет на скорость и качество работы команды.

## Принципы деловой переписки

### 1. Pyramid Principle (Барбара Минто)

**Вывод — сразу. Детали — потом.**

```
  Плохо: «Вчера мы обсуждали интеграцию, потом позвонили в поддержку,
         выяснили, что API не поддерживает формат, сейчас ищем альтернативу…
         Короче, срок сдвигается на неделю.»

  Хорошо: «Срок сдвигается на неделю (до 20 марта).
           Причина: API контрагента не поддерживает наш формат.
           Решение: разрабатываем адаптер, оценка — 5 дней.
           Альтернатива: использовать формат контрагента (минус — теряем историю).»
```

### 2. BLOT (Bottom Line On Top)

Первое предложение — суть. Далее — обоснование.

### 3. SCQA (Situation — Complication — Question — Answer)

| Элемент | Описание | Пример |
|---------|----------|--------|
| **S**ituation | Как есть | «Сейчас логин через email и пароль» |
| **C**omplication | Что не так | «50% пользователей не могут вспомнить пароль» |
| **Q**uestion | Что нужно решить | «Как упростить вход?» |
| **A**nswer | Предлагаемое решение | «Войти по SMS / Google / биометрии» |

## Типы писем аналитика

### Протокол встречи

```
Тема: Протокол встречи от 15.03 — Требования к каталогу
─────────────────────────────────────────────────────────

Участники: Иванов, Петрова, Сидоров

Решения:
  1. Каталог — иерархический (3 уровня: кат → подкат → товар)
  2. Поиск — по названию и артикулу

Action Items:
  - Иванов: API контрагента к 18.03
  - Петрова: прототип каталога к 20.03

Следующая встреча: 22.03, 11:00
```

### Запрос информации

```
Тема: Нужна информация по интеграции с CRM
──────────────────────────────────────────────

Коллеги, для подготовки ТЗ нужно узнать:

1. Какие данные передаются при создании заказа?
2. Нужна ли двухсторонняя синхронизация?
3. Какая частота запросов ожидается?

Срок: до 18.03 (среда)
```

### Согласование

```
Тема: Согласуйте SRS — модуль «Каталог» v0.3
─────────────────────────────────────────────────

Коллеги, на согласование:

[SRS-каталог-v0.3.pdf]

Основные изменения с v0.2:
  - Уточнён поиск по артикулу
  - Добавлены NFR (p95 < 500 мс)

Жду approval или комментарии до 20.03.
Без ответа — считаю согласованным.
```

## Каналы коммуникации

| Канал | Для чего | Чего избегать |
|-------|----------|---------------|
| **Email** | Формальные запросы, согласования, внешние контрагенты | Срочные вопросы |
| **Slack / Telegram** | Быстрые вопросы, уточнения | Обсуждение решений без контекста |
| **Confluence** | Документация, спецификации | Живое обсуждение |
| **Jira** | Задачи, баги, CR | Личное общение |
| **Видеозвонок** | Сложные обсуждения, конфликты | Ежедневный статус (хватит чата) |

## Типичные ошибки

- **Слишком длинные письма** — никто не дочитывает (пирамида!)
- **Письмо вместо задачи** — «ну мы же писали в чате» — нет, создайте задачу
- **Эмоции** — никогда не пишите в гневе (написали — отложите на 15 минут)
- **Нет action items** — обсудили, а что делать — неясно
- **Некорректный тон** — «вы должны», «вы обязаны» → «предлагаю», «пожалуйста»
