# Техники воркшопов

Воркшоп — это интенсивная групповая встреча для **создания артефакта** (диаграмма, roadmap, список требований) за ограниченное время.

## Фасилитация vs Воркшоп

| | Фасилитация | Воркшоп |
|--|------------|---------|
| **Цель** | Провести встречу эффективно | Создать конкретный артефакт |
| **Результат** | Решение, протокол | Модель, Roadmap, Specification |
| **Длительность** | 30 мин — 2 часа | 2 часа — 2 дня |
| **Подготовка** | Повестка | Полный сценарий, материалы |

## Типы воркшопов для аналитика

### 1. Requirements Workshop

**Цель:** собрать и согласовать требования за одну встречу.

**Когда:** сложный проект, много стейкхолдеров.

**Формат:**
1. Контекст и цели (15 мин)
2. Генерация user stories (30 мин — silent + group)
3. Приоритизация MoSCoW (20 мин — dot voting)
4. Детализация топ-5 (30 мин)
5. Следующие шаги (10 мин)

### 2. Process Discovery Workshop

**Цель:** смоделировать текущий (as-is) процесс.

**Когда:** нужно понять, как работает бизнес сейчас.

**Формат:**
1. Walkthrough процесса (участники шаг за шагом описывают)
2. Рисование модели на стикерах / Miro
3. Горячие точки (проблемы, узкие места)
4. To-be (как должно быть)

### 3. Design Sprint (адаптированный)

**Цель:** за 2 дня спроектировать решение и проверить на пользователях.

**Дни:**
1. **Day 1 — Understand + Diverge** — карта, интервью, sketches
2. **Day 2 — Decide + Prototype + Test** — решение, прототип, тест

### 4. Retrospective Workshop

**Цель:** команда ищет улучшения процессов.

**Формат:** Start / Stop / Continue

| Start (начать) | Stop (перестать) | Continue (продолжить) |
|---------------|-----------------|----------------------|
| Писать AC до разработки | Долгие daily standups | Code review |
| Тестировать на staging | ишный чат | Ретро раз в 2 недели |

## Этапы воркшопа

### 1. Preparation (подготовка)

- **Цель** — зафиксирована и согласована с заказчиком
- **Повестка** — расписана по минутам
- **Участники** — подтвердили участие
- **Материалы** — стикеры, флипчарт, Miro board, проектор
- **Email перед воркшопом** — контекст, цель, что от них нужно

### 2. Execution (проведение)

| Время | Активность | Материалы |
|-------|-----------|-----------|
| 09:00 | Opening (цель, повестка, правила) | Презентация |
| 09:15 | Сбор контекста | Флипчарт |
| 09:45 | Silent brainstorming | Стикеры |
| 10:15 | Группировка и обсуждение | Магнитная доска |
| 11:00 | Break | — |
| 11:15 | Приоритизация | Sticker dots |
| 11:45 | Детализация топ-3 | Miro |
| 12:30 | Closing, next steps | Email-протокол |

### 3. Follow-up (после)

- Протокол в течение 24 часов
- Опубликовать артефакты (диаграммы, фото доски)
- Action items с ответственными и сроками

## Инструменты для воркшопов

| Инструмент | Для чего | Когда |
|-----------|----------|-------|
| **Miro** | Онлайн-доска, стикеры, шаблоны | Удалённый воркшоп |
| **Milanote** | Визуальные доски | Дизайн-спринты |
| **Figma** | Прототипы | Design Sprint |
| **Stickies + wall** | Стикеры и стена | Офлайн (лучший вариант!) |
| **Timer (Toggl / TimeTimer)** | Тайминг | Любой воркшоп |

## Тайминг — главный навык

Правило: **лучше закончить раньше, чем не успеть.**

- На каждый пункт — запас 20% времени
- Если обсуждение ушло в детали — «паркуем» в parking lot (отдельный лист)
- Жёсткий break — 5-10 минут каждый час
- Закрытие строго по времени

## Типичные ошибки

- **Слишком много участников** — > 10 человек → разбивайте на группы
- **Нет контекста** — участники не знают, зачем пришли
- **Нет результата** — поговорили, но ничего не создали
- **Нет follow-up** — через неделю никто не помнит, что решили
- **Фасилитатор — он же участник** — невозможно вести и участвовать
