# Sequence diagram

:::note
Sequence diagram (диаграмма последовательности) показывает, в каком порядке объекты системы обмениваются сообщениями. Это сценарий взаимодействия, развёрнутый во времени.
:::

Вспомните, как вы заказываете пиццу. Вы звоните в пиццерию, оператор принимает заказ, передаёт его на кухню, повар готовит, курьер везёт. Если нарисовать это в виде временной последовательности — кто кому что говорит и в каком порядке — получится Sequence diagram.

## Основные элементы

**Линия жизни (Lifeline)** — вертикальная пунктирная линия под каждым участником. Показывает, что участник существует во времени.

**Активация (Execution occurrence)** — узкий прямоугольник на линии жизни. Означает, что участник в этот момент занят выполнением действия.

**Сообщение (Message)** — стрелка между линиями жизни. Показывает, какая информация передаётся. Стрелка от А к Б означает, что А отправляет сообщение Б.

**Типы сообщений:**
- `синхронное` — сплошная стрелка с заполненным треугольником. Отправитель ждёт ответа.
- `асинхронное` — сплошная линия с открытой стрелкой. Отправитель не ждёт, продолжает работу.
- `возврат` — пунктирная стрелка. Ответ на предыдущее сообщение.

**Фрейм (Interaction frame)** — прямоугольник, который группирует фрагменты диаграммы. Используется с операторами: `alt` (альтернатива), `opt` (опция), `loop` (цикл), `ref` (ссылка на другую диаграмму).

## Пример

Система: «Заказ такси».

Участники: Клиент, Приложение, Водитель.

```
Клиент → Приложение: "Заказать такси"
Приложение → Приложение: Найти ближайших водителей
alt водитель найден
    Приложение → Водитель: "Новый заказ"
    Водитель → Приложение: "Принять заказ"
    Приложение → Клиент: "Водитель назначен"
else водитель не найден
    Приложение → Клиент: "Нет свободных машин"
end
```

Этот сценарий читается сверху вниз: клиент отправляет запрос, приложение ищет водителя, и в зависимости от результата выполняются разные ветки.

## Как читать диаграмму

Диаграмма читается сверху вниз — от первого сообщения к последнему. Время течёт вниз. Участники расположены слева направо, обычно в порядке инициации взаимодействия. Если видите `alt` — значит, возможны варианты. `loop` — действие повторяется. `opt` — действие может не произойти.

## Когда использовать

- Чтобы показать протокол взаимодействия между компонентами.
- Для детализации конкретного use case или сценария.
- При проектировании API — показать последовательность запросов и ответов.
- Для выявления узких мест и лишних шагов.

## Когда НЕ использовать

- Для описания бизнес-процессов с ветвлениями, ролями и исключениями — здесь лучше BPMN.
- Для показа структуры системы или связей между объектами — нужна Class diagram.

## Почему это важно для аналитика

Sequence diagram — лучший способ показать разработчику, как именно система должна себя вести в конкретном сценарии. Вы описываете не «что система делает», а «в каком порядке что происходит и кто за что отвечает». Это снимает огромное количество вопросов на этапе разработки.

## Ключевые термины

- **Линия жизни** — временная ось участника.
- **Сообщение** — передача данных или команды.
- **Активация** — период, когда участник занят обработкой.
- **Фрейм** — блок с условием (alt, opt, loop).

## Что дальше

Sequence diagram часто дополняет [BPMN](/docs/modeling/bpmn) на уровне детализации шагов. Также используется при проектировании интеграций — см. [Основы REST API](/docs/integration/api-rest-basics).

## Проверь себя

1. Чем синхронное сообщение отличается от асинхронного?
2. В каком случае вы выберете Sequence diagram, а в каком — BPMN?
3. Нарисуйте Sequence diagram для сценария «Клиент возвращает товар в интернет-магазине».

**Ответы:**
1. Синхронное: отправитель ждёт ответ и блокируется. Асинхронное: отправитель отправляет сообщение и продолжает заниматься своими делами.
2. Sequence — когда нужно показать взаимодействие компонентов/систем во времени (как именно они общаются). BPMN — когда нужно показать бизнес-процесс с ролями, ветвлениями, исключениями.
3. Клиент → Магазин: «Запрос на возврат». Магазин → Клиент: «Номер возврата». Клиент → Курьер: «Передать товар». Курьер → Магазин: «Товар получен». Магазин → Клиент: «Возврат оформлен». Магазин → Банк: «Возврат средств».
