# Что такое операционная система

:::note
Операционная система (ОС) — это программа, которая управляет всеми ресурсами компьютера: запускает другие программы, распределяет память, работает с файлами и подключает устройства. Без ОС компьютер — просто набор железа, которое не умеет ничего само по себе.
:::

Представьте многоквартирный дом. В доме есть лифты, водопровод, электричество, мусоропровод. Жильцы (программы) хотят пользоваться этими ресурсами, но если все одновременно включат воду — напора не хватит.

**Управдом (операционная система)** регулирует:
- кто и когда может использовать лифт (процессор)
- сколько воды выделить каждой квартире (память)
- чтобы квартиры не мешали друг другу (изоляция процессов)
- чтобы работали мусоропровод, почта, отопление (драйверы устройств)

## Зачем нужна ОС

Без ОС каждый программист должен был бы сам писать код для работы с клавиатурой, экраном, дисками и сетью. ОС даёт **стандартный интерфейс** — программы просто говорят «открой файл» или «выведи на экран», а ОС делает всю грязную работу.

Основные функции ОС:

### Управление процессами

Вы открыли браузер, Word и Telegram одновременно. ОС решает, какой программе дать процессор прямо сейчас. Создаётся иллюзия, что всё работает одновременно (на самом деле процессор переключается между программами очень быстро).

### Управление памятью

Каждая программа думает, что у неё вся память компьютера. ОС следит, чтобы программы не залезали в память друг друга. Если программа выдает ошибку — ОС её завершит, но остальные продолжат работу.

### Файловая система

ОС организует данные на диске в виде файлов и папок. Вы видите красивую структуру «Документы → Работа → Отчёт.docx», а на самом деле файл может быть разбросан по разным секторам диска — ОС скрывает эту сложность.

### Драйверы устройств

Принтеры, сканеры, видеокарты, Wi-Fi-адаптеры — каждое устройство требует своего языка общения. ОС через **драйверы** переводит стандартные команды на язык конкретного устройства.

### Пользовательский интерфейс

Графический интерфейс (GUI) — иконки, окна, кнопки — тоже часть ОС. Вы можете не помнить команды — просто кликаете мышкой.

## Основные ОС

### Windows
Самая распространённая на десктопах. Знакома большинству пользователей. В Enterprise-среде — стандарт для рабочих станций.

### macOS
Unix-подобная ОС от Apple. Популярна среди разработчиков и дизайнеров. Основана на BSD Unix.

### Linux
Семейство Unix-подобных ОС с открытым кодом. Используется на серверах (90%+ интернета работает на Linux), в Android, в корпоративных системах. Множество дистрибутивов: Ubuntu, Debian, CentOS, Alpine.

### Мобильные ОС
- **Android** — на базе Linux, открытая экосистема
- **iOS** — на базе Darwin (Unix), закрытая экосистема Apple

## Почему это важно для аналитика

ОС — это **платформа**, на которой работает система, которую вы анализируете:

- **Совместимость**: «Работает на Windows 10+» — типичное нефункциональное требование. Если в компании используют macOS, а вы пишете под Windows — проблема
- **Ограничения**: файловые пути, права доступа, кодировки — всё зависит от ОС
- **Серверная среда**: если система будет работать на Linux — это влияет на выбор стека технологий, способ деплоя, формат логов
- **Контейнеризация**: Docker использует ядро Linux. Если ОС хоста — Windows, для Linux-контейнеров нужна виртуализация

## Ключевые термины

| Термин | Значение |
|--------|---------|
| **Ядро (kernel)** | Центральная часть ОС, управляющая ресурсами |
| **Процесс** | Запущенная программа со своей памятью |
| **Драйвер** | Модуль для работы с конкретным устройством |
| **Файловая система** | Способ организации данных на диске (NTFS, ext4, APFS) |
| **API ОС** | Системные вызовы — как программы общаются с ОС |
| **Виртуальная память** | Механизм, позволяющий программам использовать больше памяти, чем физически есть |

## Проверь себя

**1. Чем ОС отличается от прикладной программы?**

ОС управляет ресурсами компьютера и предоставляет сервисы другим программам. Прикладная программа (Word, браузер) решает конкретную задачу пользователя, используя ресурсы через ОС.

**2. Почему программа, написанная для Windows, может не работать на Linux?**

Разные API ОС, разные файловые системы, разный формат исполняемых файлов. ОС предоставляют разные «языки» для системных вызовов.

**3. Зачем аналитику знать, на какой ОС будет работать система?**

Это влияет на требования к совместимости, стоимость разработки, выбор технологий и условия эксплуатации.
