# Основы баз данных — обзорно

:::note
База данных (БД) — это организованное хранилище данных, которое позволяет эффективно сохранять, искать и изменять информацию. Большинство IT-систем работают с базами данных — это их «долговременная память».
:::

Представьте библиотеку. Книги расставлены по полкам (таблицам), у каждой книги есть уникальный номер (ID), есть каталог (индекс), и библиотекарь (СУБД) может найти любую книгу за секунды.

Теперь представьте, что все книги просто свалены в одну кучу в подвале. Найти нужную за разумное время невозможно. База данных — это и есть «библиотека» с полками, каталогом и библиотекарем.

## Что такое СУБД

СУБД (Система Управления Базами Данных, или DBMS) — это программа, которая управляет базой данных. Именно СУБД обеспечивает:

- **Хранение** — запись данных на диск в оптимальном формате
- **Поиск** — быстрый доступ по любым критериям (через индексы)
- **Изменение** — добавление, обновление, удаление записей
- **Целостность** — защита от потери данных при сбоях
- **Безопасность** — разграничение доступа (кто что может читать и писать)

Популярные СУБД: PostgreSQL, MySQL, Microsoft SQL Server, Oracle, SQLite.

## Реляционные базы данных

Самый распространённый тип БД. Данные организованы в виде **таблиц** со строками и столбцами.

**Автомобили**:

| ID | Марка | Модель | Год | Цвет |
|----|-------|--------|-----|------|
| 1 | Toyota | Camry | 2020 | Синий |
| 2 | BMW | X5 | 2022 | Чёрный |
| 3 | Lada | Vesta | 2023 | Белый |

### Таблицы и связи

Данные распределяются по нескольким таблицам, связанным между собой. Например, таблица «Заказы» связана с таблицей «Клиенты» через ID клиента:

**Клиенты**:
| ID | Имя | Телефон |
|----|-----|---------|
| 1 | Иван | +7-999... |

**Заказы**:
| ID | КлиентID | Товар | Сумма |
|----|----------|-------|-------|
| 101 | 1 | Ноутбук | 75000 |

Такая организация позволяет не повторять данные (адрес Ивана хранится один раз, а не в каждом заказе).

### Что такое SQL

SQL (Structured Query Language) — язык для работы с реляционными БД. С его помощью можно:

- **SELECT** — найти данные (SELECT * FROM Клиенты WHERE Город='Москва')
- **INSERT** — добавить запись
- **UPDATE** — изменить данные
- **DELETE** — удалить данные

Аналитику не нужно писать сложные SQL-запросы, но понимать базовый синтаксис полезно — чтобы проверить данные в тестовой среде или понять, что разработчик имеет в виду под «сложным join'ом».

## Другие типы БД

### NoSQL

Базы данных, которые не используют реляционную модель. Хороши для специфических задач:

- **MongoDB** — хранит данные в виде JSON-документов (гибкая структура)
- **Redis** — хранит данные в памяти (очень быстро, для кэша)
- **Elasticsearch** — для полнотекстового поиска
- **Neo4j** — графовая БД (для связей между сущностями)

### Когда что выбирать

| Задача | Тип БД | Пример |
|--------|--------|--------|
| Банковская система, учёт | Реляционная | PostgreSQL |
| Каталог товаров (разные атрибуты у товаров) | Документная | MongoDB |
| Кэш, сессии пользователей | Key-Value | Redis |
| Поиск по тексту | Поисковая | Elasticsearch |
| Соцсеть, рекомендации | Графовая | Neo4j |

## Почему это важно для аналитика

Данные — основной объект работы аналитика:

- **Требования** к функциональности почти всегда связаны с данными (что сохранять, как искать, какие отчёты строить)
- **Моделирование данных** — часть работы аналитика (ER-диаграммы, описание сущностей)
- **Интеграции** между системами — это обмен данными, часто через БД
- **SQL** нужен, чтобы проверять данные, анализировать логи, готовить отчёты

Типичные вопросы аналитика к БД:
- «Покажите структуру таблиц, чтобы я понимал, какие данные у нас есть»
- «Мне нужно описать новую сущность — где она будет храниться?»
- «Сможем ли мы искать заказы по дате и статусу одновременно?»

## Ключевые термины

| Термин | Значение |
|--------|---------|
| **БД (База данных)** | Организованное хранилище данных |
| **СУБД (DBMS)** | Система управления базами данных (программа) |
| **Таблица** | Набор связанных записей со столбцами |
| **Строка / Запись** | Один элемент данных в таблице |
| **Столбец / Поле** | Один атрибут записи |
| **Первичный ключ (PK)** | Уникальный идентификатор записи |
| **Внешний ключ (FK)** | Ссылка на запись в другой таблице |
| **SQL** | Язык запросов к реляционным БД |
| **Индекс** | Структура для ускорения поиска |

## Проверь себя

**1. Зачем данные хранят в нескольких таблицах, а не в одной?**

Чтобы избежать дублирования (нормализация). Если данные клиента хранятся в каждой строке заказа, при смене адреса придётся обновлять тысячи строк. Если хранить отдельно — достаточно изменить одну запись в таблице клиентов.

**2. Чем СУБД отличается от файла Excel?**

СУБД обеспечивает целостность данных (не даст удалить клиента, у которого есть заказы), одновременный доступ сотен пользователей, поиск за миллисекунды, защиту от сбоев. Excel — это файл, с которым работает один человек.

**3. Зачем аналитику понимать разницу между PostgreSQL и MongoDB?**

Потому что это влияет на архитектуру системы. Если данные строго структурированы и важна целостность — нужна реляционная БД. Если структура гибкая и меняется — документная БД. Неверный выбор приводит к проблемам на этапе разработки.
