# HTTP — протокол передачи данных

HTTP (HyperText Transfer Protocol) — это протокол прикладного уровня, лежащий в основе передачи данных в Вебе. Понимание HTTP необходимо любому системному аналитику, работающему с веб-системами.

## Структура HTTP-запроса

Каждый HTTP-запрос состоит из трёх частей:
- **Стартовая строка** — метод, URI, версия протокола
- **Заголовки (headers)** — метаданные запроса
- **Тело (body)** — передаваемые данные (необязательно)

### Методы HTTP

Наиболее часто используемые методы:

| Метод   | Назначение                     | Идемпотентность |
|---------|--------------------------------|-----------------|
| GET     | Получение ресурса              | Да              |
| POST    | Создание ресурса               | Нет             |
| PUT     | Полная замена ресурса          | Да              |
| PATCH   | Частичное обновление ресурса   | Нет             |
| DELETE  | Удаление ресурса               | Да              |

## Коды состояния

HTTP-ответы группируются по классам:
- **1xx** — информационные (редко используются)
- **2xx** — успех (200 OK, 201 Created, 204 No Content)
- **3xx** — перенаправление (301 Moved, 304 Not Modified)
- **4xx** — ошибка клиента (400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found)
- **5xx** — ошибка сервера (500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable)

## Версии HTTP

HTTP/1.1 — самая распространённая версия, поддерживает keep-alive соединения. HTTP/2 — добавляет мультиплексирование и сжатие заголовков. HTTP/3 — работает поверх QUIC вместо TCP для снижения задержек.

## Практическое применение

При проектировании API системный аналитик должен:
1. Выбирать правильные HTTP-методы для операций
2. Предусматривать все возможные коды ответа
3. Проектировать заголовки для аутентификации и кэширования
4. Учитывать ограничения протокола (размер тела, таймауты)
