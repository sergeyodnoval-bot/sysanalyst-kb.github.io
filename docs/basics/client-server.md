# Клиент-серверная архитектура

Клиент-серверная архитектура — это модель распределённой системы, в которой нагрузка по обработке данных разделена между поставщиком услуг (сервером) и потребителями услуг (клиентами).

## Основные понятия

**Сервер** — это программа или устройство, которое предоставляет ресурсы или услуги другим программам (клиентам). Сервер обычно работает непрерывно и ожидает входящих запросов.

**Клиент** — это программа, которая инициирует соединение с сервером и отправляет запросы на получение или изменение данных.

### Принцип работы

1. Клиент формирует запрос
2. Отправляет его по сети на сервер
3. Сервер обрабатывает запрос
4. Сервер отправляет ответ клиенту
5. Клиент обрабатывает ответ

## Преимущества модели

- **Масштабируемость** — можно добавлять серверы горизонтально
- **Централизованное управление** — данные хранятся на сервере
- **Безопасность** — доступ к данным контролируется на сервере
- **Тонкий клиент** — на стороне клиента не требуется много ресурсов

## Типы клиент-серверных архитектур

### Двухзвенная архитектура

Клиент напрямую обращается к серверу. Пример: веб-браузер и веб-сервер.

### Трёхзвенная архитектура

Добавляется промежуточный слой — сервер приложений, который содержит бизнес-логику:

```
Клиент → Сервер приложений → Сервер БД
```

### Многоуровневая архитектура (N-tier)

Разделение на множество слоёв: презентационный, бизнес-логика, интеграционный, доступа к данным.

## Роль системного аналитика

При проектировании клиент-серверной архитектуры аналитик должен:
- Определить границы между компонентами
- Спроектировать интерфейсы взаимодействия
- Выбрать протоколы обмена данными
- Документировать архитектурные решения
