:::info[TL;DR]
Спроектировать систему доставки последней мили для интернет-магазина: слоты доставки, назначение курьеров, трекинг, подтверждение получения. Результат: статусная модель, sequence diagram, спецификация API.
:::

## Контекст

Интернет-магазин (1000+ заказов/день) запускает собственную доставку в городе-миллионнике. Нужно спроектировать систему: слоты, курьеры, трекинг.

## Что нужно сделать

1. Описать статусную модель заказа доставки (8+ статусов)
2. Спроектировать слоты доставки (2-часовые окна, ограничения)
3. Нарисовать sequence diagram: заказ → назначение курьера → маршрут → вручение
4. Описать интеграцию с геоданными (геокодинг адресов, матрица расстояний)
5. Специфицировать API для мобильного приложения курьера (получить заказы, отметить статус)
6. Описать сценарии ошибок: курьер опаздывает, клиента нет дома, отказ от заказа

## Результат

- Status model (state diagram + таблица переходов)
- Sequence diagram (3+ участника)
- API-спецификация (3+ endpoint)
- Сценарии ошибок (3+ с обработкой)

## Критерии приемки

- Статусная модель содержит 8+ состояний
- Sequence diagram отражает полный цикл доставки
- API покрывает получение, изменение статуса, трекинг
- Обработка ошибок для 3+ кейсов
