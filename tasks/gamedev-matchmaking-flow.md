:::info[TL;DR]
Спроектировать матчмейкинг для PvP-игры (1v1, 3 min, рейтинг). Описать критерии подбора, статусную модель сессии, таймауты, обработку отключений. Результат: sequence diagram, таблица критериев, спецификация API.
:::

## Контекст

Мобильная PvP-игра (1v1, матч ~3 минуты). Рейтинг ELO. Нужен быстрый и честный матчмейкинг.

## Что нужно сделать

1. Описать алгоритм подбора: MMR, ping, диапазон ожидания
2. Спроектировать статусную модель сессии (6+ состояний)
3. Нарисовать sequence diagram: игрок → очередь → матчмейкер → сессия → результат
4. Описать таймауты: max queue time (от 10s), расширение диапазона MMR
5. Спроектировать обработку: игрок отключился, вышел, не загрузился
6. Определить метрики: queue time, quality of match, abandon rate

## Результат

- Алгоритм подбора (таблица критериев)
- State diagram сессии (Mermaid)
- Sequence diagram (3+ участника)
- Таблица таймаутов (сценарий → действие)

## Критерии приемки

- 4+ критерия подбора (MMR, ping, уровень, ожидание)
- 6+ состояний сессии
- Sequence diagram включает обработку ошибок
- Таймауты и их расширение описаны
