## Контекст

Вы — системный аналитик в e-commerce компании. Маркетинг хочет запустить программу лояльности: баллы + уровни + реферальная программа. Нужно спроектировать систему.

**Требования:**
- Баллы начисляются за покупки (N% от суммы)
- Есть 4 уровня (Silver → Gold → Platinum → VIP)
- Баллы можно тратить на оплату до 50% заказа
- Реферальная программа: +500 баллов за друга
- Баллы сгорают через 365 дней без активности
- Система должна интегрироваться с существующей OMS

## Что нужно сделать

1. **Правила начисления и списания** — описать формулы для каждого уровня, лимиты, исключения
2. **Схема данных** — ER-диаграмма (customers, loyalty_accounts, transactions, tiers, referrals)
3. **API-спецификация** — эндпоинты для работы с лояльностью (баланс, начисление, списание, история)
4. **Интеграция с OMS** — sequence diagram: что происходит, когда заказ оплачен
5. **Cron-задачи** — какие задачи по расписанию (сгорание баллов, пересчёт уровня)

## Критерии приемки

- [ ] Правила покрывают все 4 уровня с разными rate и привилегиями
- [ ] ER-диаграмма содержит не менее 5 таблиц со связями
- [ ] API-спецификация содержит не менее 5 эндпоинтов
- [ ] Sequence diagram покрывает happy path (оплата → начисление) + error path (возврат → списание)
- [ ] Описаны cron-задачи с периодичностью и логикой
