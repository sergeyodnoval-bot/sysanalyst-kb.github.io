# Проектирование AI-агента для продакшна

Спроектируйте AI-агента для автоматизации технической поддержки.

## Контекст

Компания «ТехноПоддержка» хочет внедрить AI-агента, который будет отвечать на входящие запросы клиентов. Текущий процесс:

1. Клиент пишет в чат или оставляет заявку через форму
2. Оператор классифицирует запрос (баг, вопрос, жалоба)
3. Оператор ищет ответ в базе знаний
4. Если ответа нет — передаёт тикет разработчикам

## Требования

- Агент должен отвечать на 70% типовых запросов без участия человека
- Время ответа — не более 30 секунд
- Агент должен иметь доступ к базе знаний (Confluence) и системе тикетов (Jira)
- Сложные/неоднозначные запросы передаются человеку
- Все действия агента логируются
- Бюджет на AI-инфраструктуру — $500/мес

## Задачи

1. **Архитектура**
   - Выберите паттерн (один агент vs мультиагент)
   - Опишите схему: какие компоненты, какие MCP-серверы, как организована коммуникация

2. **Спецификация скилов**
   - Опишите 3–4 скила агента (структура: инструменты, ограничения, критерии качества)

3. **Инструменты (MCP)**
   - Какие MCP-серверы нужны? Какие инструменты они предоставляют?
   - Уровни доступа: что read-only, что read-write

4. **Безопасность и guardrails**
   - Какие действия агент не должен выполнять?
   - Как защититься от prompt injection?

5. **Метрики качества**
   - Определите success rate, escalation rate, average cost per ticket
   - Какие пороги считаете приемлемыми?

## Результат

Документ (Markdown) из 5 секций. Укажите trade-off'ы и обоснуйте решения.
