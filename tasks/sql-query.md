# Написать SQL-запрос по ТЗ

**Контекст:** Product Owner хочет получить данные о клиентах. Вам нужно написать корректный SQL-запрос к реальной БД (или тестовой) и показать результат. Это типичная задача Junior аналитика — данные нужны часто, а разработчики не всегда успевают.

## Цель задачи

Написать SQL-запрос, который возвращает именно те данные, которые запросил заказчик. Без лишних строк, без дублей, с правильной фильтрацией.

## Пошаговый подход

### Шаг 1. Изучите схему БД

Подключитесь к БД через DBeaver. Найдите нужные таблицы. Какие поля есть? Как они связаны?

Пример схемы для интернет-магазина:

| Таблица | Поля |
|---------|------|
| `customers` | id, name, email, status (active/inactive), created_at |
| `orders` | id, customer_id, total, status, created_at |
| `returns` | id, order_id, reason, created_at |

Связи:
- `customers.id` → `orders.customer_id` (один ко многим)
- `orders.id` → `returns.order_id` (один ко многим)

### Шаг 2. Определите нужные таблицы

Для запроса «активные клиенты с заказом за 3 месяца, без возврата»:

- `customers` — для полей клиента и фильтра `status = 'active'`
- `orders` — для проверки наличия заказа за последние 3 месяца
- `returns` — для исключения клиентов с возвратами

### Шаг 3. Напишите запрос

Начните с базового, добавляйте по одному условию:

**Шаг 3.1. Все активные клиенты:**
```sql
SELECT id, name, email
FROM customers
WHERE status = 'active';
```

**Шаг 3.2. Клиенты с заказами за 3 месяца:**
```sql
SELECT DISTINCT c.id, c.name, c.email
FROM customers c
JOIN orders o ON o.customer_id = c.id
WHERE c.status = 'active'
  AND o.created_at >= NOW() - INTERVAL '3 months';
```

**Шаг 3.3. Исключить клиентов с возвратами:**
```sql
SELECT DISTINCT c.id, c.name, c.email
FROM customers c
JOIN orders o ON o.customer_id = c.id
WHERE c.status = 'active'
  AND o.created_at >= NOW() - INTERVAL '3 months'
  AND c.id NOT IN (
    SELECT DISTINCT o2.customer_id
    FROM returns r
    JOIN orders o2 ON o2.id = r.order_id
  );
```

### Шаг 4. Выполните и проверьте

1. Откройте DBeaver
2. Выберите вашу БД
3. Вставьте запрос в редактор SQL
4. Выполните (Ctrl+Enter или F5)
5. Посмотрите на результат: правильное количество строк? Те поля, что ожидались?

### Шаг 5. Опишите логику

Напишите кратко для заказчика:

> Запрос выбирает активных клиентов, у которых есть хотя бы один заказ за последние 3 месяца, и при этом ни один из их заказов не возвращался. Используется LEFT JOIN для orders и исключение через NOT IN для клиентов с возвратами. Результат: 42 клиента.

## Критерии выполнения

- [ ] Запрос возвращает корректные данные (проверено на реальной БД)
- [ ] Нет дублей (использован DISTINCT или GROUP BY)
- [ ] Выбраны только нужные поля (не SELECT *)
- [ ] Учтены NULL-значения
- [ ] Логика запроса описана простыми словами

## Типичные ошибки

**Не смотреть схему.** Самый частый косяк — писать запрос «вслепую», угадывая названия полей. Откройте DBeaver, посмотрите структуру таблиц. Там могут быть неочевидные поля или связи.

**Забыть про DISTINCT.** При JOIN один клиент может иметь 5 заказов → в результате будет 5 строк с одинаковыми данными клиента. Если нужен список клиентов (без дублей) — используйте DISTINCT.

**SELECT \*.** Если таблица customers имеет 20 полей, заказчику скорее всего нужны только 3-4. Выбирайте явно нужные поля.
