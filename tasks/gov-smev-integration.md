## Контекст

Вы — системный аналитик в ведомстве, которое разрабатывает новую ГИС для выдачи лицензий. Эта ГИС должна интегрироваться с СМЭВ 3 для получения данных из других госорганов.

**Требования:**
- Транспорт: REST / HTTP/2
- Аутентификация: OAuth 2.0 + УКЭП
- Формат: JSON
- Нужно получать: ИНН (ФНС), СНИЛС (СФР), выписку ЕГРЮЛ (ФНС)

## Что нужно сделать

1. **Схема интеграции** — описать взаимодействие ГИС → СМЭВ → ГИС-поставщик
2. **Модель данных** — структура запроса/ответа для каждого вида сведений
3. **SLA и таймауты** — для каждого запроса указать таймаут и ретраи
4. **Ошибки** — какие ошибки могут быть (таймаут, отказ, неверные данные)
5. **Логирование и аудит** — какие события нужно логировать для ФСТЭК

## Критерии приемки

- [ ] Схема интеграции содержит не менее 5 участников (ГИС, шлюз, СМЭВ, поставщик)
- [ ] Для каждого вида сведений (3 шт.) — JSON-структура запроса и ответа
- [ ] SLA: таймаут указан для каждого запроса
- [ ] Ошибки: не менее 4 сценариев ошибок с описанием реакции системы
- [ ] Аудит: перечень логируемых событий (не менее 5)
