## Контекст

Вы — системный аналитик в организации, которая разработала ГИС для выдачи справок гражданам. Система обрабатывает ПД (ФИО, паспорт, СНИЛС). Нужно подготовить чек-лист для аттестации по ФСТЭК.

**Известные данные:**
- Пользователи: 500 операторов, 50 администраторов
- Данные: ПД (УЗ-1), объём — 10 ТБ
- Инфраструктура: Astra Linux, Postgres Pro, КриптоПро
- Интеграции: СМЭВ (3 внешних сервиса)

## Что нужно сделать

1. **Категорирование** — определить класс защищённости (УЗ-1, УЗ-2, УЗ-3)
2. **Меры защиты** — выбрать меры из приказа ФСТЭК № 239 по каждой группе (ИАФ, УПД, ОЛС, ЗИС, ЗНИ, ЗСВ, РСБ, АВЗ)
3. **Модель угроз** — описать источники угроз для этой ГИС
4. **Документация** — перечень документов для аттестации
5. **Организационные меры** — приказы, инструкции, регламенты

## Критерии приемки

- [ ] Класс защищённости определён и обоснован
- [ ] Выбраны меры защиты по каждой группе (минимум 8 мер)
- [ ] Модель угроз содержит не менее 5 типов угроз
- [ ] Перечень документов: не менее 10 позиций
- [ ] Организационные меры: не менее 3 (приказы/инструкции/регламенты)
