# Заведение баг-репорта

**Контекст:** Заказчик сообщает, что кнопка «Оформить заказ» не работает. Вы — аналитик, который отвечает за качество требований и коммуникацию с командой. Баг-репорт — ваш инструмент, чтобы донести проблему до разработчиков без потери деталей.

## Цель задачи

Завести качественный баг-репорт, по которому разработчик сможет гарантированно воспроизвести и исправить проблему.

## Пошаговый подход

### Шаг 1. Воспроизведите баг

Прежде чем писать репорт — убедитесь, что баг воспроизводится. Попробуйте сделать то же самое, что делал заказчик. Если не получается — уточните.

### Шаг 2. Определите окружение

Запишите все детали окружения:

| Параметр | Пример |
|----------|--------|
| Браузер | Chrome 124, Firefox 125 |
| ОС | Windows 11, macOS 14 |
| Устройство | Desktop, iPhone 15 |
| Версия приложения | v2.3.1 |
| Роль пользователя | Менеджер (не админ) |

### Шаг 3. Опишите шаги воспроизведения

Каждый шаг — конкретное действие пользователя:

1. Открыть страницу https://example.com/orders
2. Авторизоваться под учётной записью менеджера (login: manager@test.com)
3. Нажать кнопку «Оформить заказ» на заказе №12345
4. Наблюдать результат

### Шаг 4. Укажите ожидаемый и фактический результат

**Ожидаемый результат:** Открывается форма оформления заказа с полями: адрес доставки, способ оплаты, комментарий.

**Фактический результат:** Ничего не происходит. Кнопка не реагирует на нажатие. В консоли ошибок нет.

### Шаг 5. Приложите скриншот/видео

Скриншот показывающий кнопку до и после нажатия. Или запись экрана с воспроизведением.

### Шаг 6. Определите приоритет

| Приоритет | Критерий | Пример |
|-----------|----------|--------|
| Blocker | Невозможно работать с системой | Падение сервера, потеря данных |
| Critical | Ключевая функция не работает | Не работает оформление заказа |
| Major | Функция работает с ошибками | Неправильный расчёт скидки |
| Minor | Некритичная ошибка | Опечатка в тексте |
| Trivial | Косметика | Съехала вёрстка на одном экране |

### Шаг 7. Создайте задачу в Jira

1. Откройте проект в Jira
2. Create Issue → тип Bug
3. Summary: «Кнопка "Оформить заказ" не реагирует на нажатие (Chrome 124)»
4. Заполните Description по шаблону выше
5. Укажите Environment: Chrome 124, Windows 11
6. Укажите Priority: Critical
7. Приложите скриншот
8. Нажмите Create

## Критерии выполнения

- [ ] Баг воспроизведён автором репорта
- [ ] Указано окружение (браузер, ОС, версия)
- [ ] Шаги воспроизведения конкретны и воспроизводимы
- [ ] Приложен скриншот или видео
- [ ] Приоритет обоснован (соответствует критериям)

## Типичные ошибки

**«Ничего не работает».** Самая бесполезная формулировка. Что именно не работает? В каком браузере? На какой странице? После каких действий? Конкретность — главное качество баг-репорта.

**Нет окружения.** Баг может воспроизводиться только в Firefox или только на iPhone. Без указания окружения разработчик не сможет его поймать.

**Нет скриншота.** Люди по-разному понимают текст. Скриншот однозначно фиксирует проблему. Видео — ещё лучше.

**Дубликат.** Перед созданием проверьте, нет ли уже такого бага. Поищите по ключевым словам в Jira.
