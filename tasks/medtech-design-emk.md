:::info[TL;DR]
Спроектировать структуру ЭМК для городской поликлиники: состав разделов, права доступа, жизненный цикл записей, интеграции с ЛИС и PACS. Результат: схема данных, use-case диаграмма, матрица доступов.
:::

## Контекст

Городская поликлиника на 500 посещений в день переходит с бумажных карт на ЭМК. Внедряется МИС. Нужно спроектировать структуру электронной медкарты.

## Что нужно сделать

1. Определить состав разделов ЭМК (демография, диагнозы, посещения, прививки, льготы)
2. Спроектировать права доступа (RBAC) для ролей: врач, медсестра, заведующий, администратор
3. Описать жизненный цикл записи в ЭМК (черновик → подписано → архив / редактирование)
4. Определить интеграционное взаимодействие: МИС ↔ ЛИС, МИС ↔ PACS, МИС ↔ ЕМИАС
5. Нарисовать use-case диаграмму (участники: пациент, врач, медсестра, администратор)

## Результат

- ER-диаграмма (сущности: Patient, Visit, Diagnosis, Prescription, Document)
- Use-case диаграмма (UML)
- Матрица доступов (роль × раздел × действие)
- Описание жизненного цикла записи (state diagram)

## Критерии приемки

- Все разделы ЭМК покрыты структурой данных
- Права доступа описаны для 4+ ролей
- Интеграции с ЛИС, PACS и ЕМИАС отражены в схеме
- State diagram содержит 4+ состояния записи
