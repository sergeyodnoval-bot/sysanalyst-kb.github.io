:::info[TL;DR]
Спроектировать интеграцию TMS с курьерской службой (СДЭК / Boxberry). Описать API-вызовы: создание заказа, трекинг, генерация этикетки, возврат. Результат: sequence diagram, спецификация API, обработка ошибок.
:::

## Контекст

Ритейлер подключает СДЭК как курьерскую службу для доставки заказов. Нужно спроектировать интеграцию TMS → СДЭК API.

## Что нужно сделать

1. Описать flow-диаграмму: TMS → создание заказа → генерация этикетки → передача курьеру → трекинг → подтверждение
2. Специфицировать API-запросы:
   - Создание заказа на доставку (адрес, вес, габариты)
   - Генерация штрихкода / этикетки
   - Получение статуса заказа
   - Создание возврата
3. Описать форматы данных (JSON, XML — какой провайдер требует)
4. Описать сценарии ошибок: неверный адрес, превышение веса, отмена заказа

## Результат

- Sequence diagram (TMS ↔ Курьерская служба)
- Спецификация 4+ API-операций
- Статусная модель (TMS статус ↔ статус курьера)
- Обработка ошибок (3+ сценария)

## Критерии приемки

- API-спецификация покрывает создание, трекинг, этикетку, возврат
- Маппинг статусов TMS ↔ курьерская служба
- Sequence diagram показывает flow end-to-end
- Ошибки обработаны (retry, timeout, невалидные данные)
