:::info[TL;DR]
Описать требования к системе оптимизации маршрутов для курьерской доставки: входные данные (заказы, адреса, временные окна), ограничения (вес, объём, время работы), бизнес-правила и метрики качества. Результат: спецификация VRP-задачи, матрица ограничений, метрики.
:::

## Контекст

Служба доставки (200 курьеров, 5000 заказов/день) хочет внедрить оптимизацию маршрутов вместо ручного планирования.

## Что нужно сделать

1. Описать бизнес-правила маршрутизации:
   - Временные окна клиентов (2-часовые слоты)
   - Вместимость курьера (вес, объём)
   - Рабочее время курьеров (смена 8 часов)
   - Приоритеты (VIP, срочные)
2. Специфицировать входные данные для VRP-солвера:
   - Список заказов (адрес, вес, объём, слот)
   - Матрица расстояний / времени между точками
   - Параметры курьеров
3. Определить метрики оптимизации:
   - Общий пробег
   - Количество заказов на курьера
   - On-time delivery rate
4. Описать ребалансировку в реальном времени (новые заказы, отмены)

## Результат

- Матрица ограничений (constraint table)
- Спецификация входных данных для VRP
- Метрики качества (3+ метрики)
- Ребалансировка (flow + правила)

## Критерии приемки

- 5+ бизнес-правил описаны
- Входные данные специфицированы (что, формат, откуда)
- 3+ метрики для оценки маршрутов
- Ребалансировка: как обрабатываются новые/отменённые заказы
