## Контекст

Вы — системный аналитик в e-commerce компании. Продуктовая команда проектирует новый каталог товаров для интернет-магазина одежды. Нужно спроектировать структуру каталога.

**Требования:**
- Товары: одежда, обувь, аксессуары
- Каждый товар имеет варианты (цвет, размер)
- Нужен поиск по названию, бренду, категории
- Фильтры: цена, цвет, размер, бренд, материал, сезон
- Остатки должны быть в реальном времени (с задержкой не более 1 мин)
- Админка: добавление товаров, массовое обновление цен через XLS
- SEO: у каждого товара должен быть уникальный URL

## Что нужно сделать

1. **Структура каталога** — описать иерархию: Category → Subcategory → Product Model → SKU. Какие атрибуты на каком уровне?
2. **Таблица атрибутов** — какие атрибуты товара обязательны, какие — опциональны, какие — вариативные (влияют на SKU)?
3. **Схема данных** — ER-диаграмма сущностей каталога (не менее 5 таблиц)
4. **API-спецификация** — эндпоинты для работы с каталогом (CRUD + поиск + фильтры)
5. **Синхронизация остатков** — описать, как остатки из WMS попадают в каталог и Elasticsearch

## Критерии приемки

- [ ] Структура каталога содержит не менее 4 уровней иерархии
- [ ] ER-диаграмма содержит не менее 5 связанных таблиц с типами полей
- [ ] API-спецификация содержит не менее 6 эндпоинтов (CRUD + search + filters)
- [ ] Описан механизм синхронизации остатков (OMS/WMS → каталог)
- [ ] Учтён сценарий: один товар имеет 10+ вариантов (цвет × размер)
